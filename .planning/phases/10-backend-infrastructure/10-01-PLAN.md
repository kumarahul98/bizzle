1. `src/types/trip.ts` — the Phase 10↔11 CONTRACT (D-10). Export:
   - `interface Trip` — camelCase fields exactly as in <interfaces> above:
     id, userId, startTime, endTime, durationSeconds, distanceMeters,
     routePolyline (string | null), direction, timeMovingSeconds,
     timeStuckSeconds, isManualEntry, createdAt, updatedAt. Timestamps are
     ISO 8601 UTC `string`s (D-10).
   - `interface TripDoc extends Trip` — adds server metadata: `deleted: boolean`,
     `deletedAt: Timestamp | null`, `serverUpdatedAt: Timestamp`. Import
     `Timestamp` from `firebase-admin/firestore`.
   - `type Direction = 'to_office' | 'to_home'` and type `direction` as `Direction`.

2. `src/utils/response.ts` (D-06) — export `sendSuccess<T>(res, statusCode, data: T)`
   and `sendError(res, statusCode, error: string)`. Each sets `res.status(statusCode)`
   and `.json({ statusCode, body: { data } })` or `{ statusCode, body: { error } }`.
   Never leak stack traces/tokens. Type `res` as `express.Response`.

3. `src/utils/auth.ts` (D-07) — verify→trust foundation. Export:
   - `class AuthError extends Error` with a `statusCode = 401`.
   - `extractBearerToken(header: string | undefined): string` — splits the
     `Authorization` header; requires the `Bearer ` scheme; throws `AuthError`
     on missing/malformed.
   - `async verifyAuth(req: express.Request): Promise<string>` — calls
     `extractBearerToken(req.headers.authorization)`, then
     `getAuth().verifyIdToken(token)` (import `getAuth` from `firebase-admin/auth`),
     returns `decoded.uid`. Catch verify failures and rethrow as `AuthError`.
   Keep token extraction a separate exported pure function so it is unit-testable
   without the Admin SDK.

4. `src/utils/validation.ts` (D-07) — zod schemas. Export:
   - `tripSchema` — z.object mirroring `Trip`. `id` non-empty string; timestamps
     validated as ISO strings (`z.string().datetime()`); `direction`
     `z.enum(['to_office','to_home'])`; `routePolyline` `z.string().nullable()`;
     `distanceMeters` `z.number()`; integer seconds fields `z.number().int()`;
     `isManualEntry` `z.boolean()`; `userId` `z.string().optional()` (ignored — D-08).
   - `syncTripsBody` — `z.object({ trips: z.array(tripSchema) })` (D-12).
   - `tripIdParam` — `z.object({ tripId: z.string().min(1) })`.
   Export inferred TS types via `z.infer` where useful.

5. `src/utils/firestore.ts` (D-09/D-10) — export:
   - `tripConverter: FirestoreDataConverter<TripDoc>` (typed `toFirestore` /
     `fromFirestore`). Import types from `firebase-admin/firestore`.
   - `tripsCollection()` — returns `getFirestore().collection('trips')
     .withConverter(tripConverter)` (top-level `trips`, doc id = trip UUID).

6. `src/index.ts` — boot wiring:
   - `import { initializeApp } from 'firebase-admin/app'; initializeApp();`
   - `import { setGlobalOptions } from 'firebase-functions/v2';
     setGlobalOptions({ region: 'us-central1' });` (D-02)
   - Construct the Express app: `const app = express(); app.use(express.json());`
   - Mount ONLY `app.get('/health', (_req, res) => res.status(200).json({ status: 'ok' }));`
     (the three /trips routes come in Plan 02).
   - `export { app };` (exported separately so Plan 02's supertest can import it
     without the Functions wrapper).
   - `export const api = onRequest(app);` (`onRequest` from
     `firebase-functions/v2/https`).
