<objective>
Make Home/Office useful for trips that already exist: when the user sets or changes a location, re-label historical trips by proximity using the SAME pure GeofenceDirectionResolver that finalize uses (Plan 01), without ever clobbering a manual label. This is the LOC-02 historical half (Plan 01 covered new trips at finalize).

Purpose: SC#2/#3/#5 for the existing history. The `direction_source` column from Plan 01 is what makes this safe — backfill touches only `!= manual` rows. The resolver is reused verbatim so live and backfilled labels are identical. Backfill must be idempotent and a no-op when nothing is set.
Output: a backfill service, a one-shot trigger provider, the DAO query/update, the picker wiring, and an exhaustive backfill test.
</objective>

<tasks>

## Task 1 — DAO query + update (TDD)
- In `trips_dao.dart`: add a query returning trips where `direction_source != kDirectionSourceManual` AND `route_polyline` is non-null/non-empty (candidates for geofence relabel); add a single-trip update writing `direction` + `direction_source`. Reuse existing companion-update style.
- Unit test the DAO query/update against an in-memory DB.

## Task 2 — GeofenceBackfillService (TDD, core correctness)
- Build `GeofenceBackfillService` (pure-ish, takes the DAO + the resolver + prefs coords): `run()` loads candidate trips, for each decodes the polyline endpoints (`polyline_codec`), calls `GeofenceDirectionResolver` (the Plan 01 service); if it returns a confident label, write `direction` + `direction_source = kDirectionSourceGeofence`; otherwise leave the row untouched.
- Tests (exhaustive): manual rows are never selected/changed; a trip whose end is within radius is relabelled + source=geofence; a non-matching trip is unchanged (label + source preserved); a no-polyline manual-entry trip is skipped; idempotent (second run = zero writes); no Home/Office set → zero candidates processed / no-op.

## Task 3 — Trigger provider + picker wiring
- `geofence_backfill_provider.dart`: a one-shot task (mirror `directionBackfillProvider` keepAlive semantics) that runs `GeofenceBackfillService.run()` when Home/Office prefs change. Decide the cleanest trigger: invalidate/refresh it from `LocationPickerScreen` right after a successful save (preferred — deterministic), and/or have it react to the prefs change. Avoid running on every prefs emission (debounce/guard so an unrelated pref change doesn't trigger a full backfill).
- Wire `LocationPickerScreen`'s confirm path to kick the backfill after persisting the coord.
- Add `kDirectionSourceGeofence` reference (already a constant from Plan 01) — no new magic strings.

</tasks>
