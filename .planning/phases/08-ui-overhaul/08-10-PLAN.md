The current notification (`TrackingNotificationService.showRecording()`) emits a plain text notification with title "Recording commute" and one Stop action. This plan replaces it with a custom-layout notification matching the Traevy design and wires the existing 1Hz snapshot stream to update the layout's stat row live.

Custom appearance (avatar + REC pill + JetBrains Mono stat line) cannot be expressed via `flutter_local_notifications`'s standard text APIs on Android — Android system notifications use the system font for `setContentText`. The plan uses Android `RemoteViews` (via `flutter_local_notifications`'s `styleInformation` with `AndroidNotificationDetails(styleInformation: ...)` — specifically `AndroidNotificationDetails.styleInformation = null` plus custom layout via the plugin's RemoteViews bridge OR fallback to standard `BigTextStyleInformation` if the bridge is unavailable). See pitfalls for the exact API path.
