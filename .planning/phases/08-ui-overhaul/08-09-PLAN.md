**Gap 2 (truth, from `08-HUMAN-UAT.md` line 64):**
> "Tracking screen SPEED tile reflects current instantaneous speed (drops to 0 when stationary)"

Per `.planning/debug/active-speed-tile-stale.md`, the bug is in the producer (`TripAccumulator.snapshot()` at `lib/features/tracking/services/trip_accumulator.dart:196`) — it emits `currentSpeedMs: _lastAccepted?.speed ?? 0` with no freshness check. Android throttles GPS emissions when stationary AND `Position.speed` is a smoothed/sticky value. The 1Hz UI snapshot timer republishes the stale value indefinitely.

This plan adds a `_lastAcceptedAt` timestamp to TripAccumulator, gates `snapshot()` to emit `0` when stale, and adds unit tests covering the three documented cases. Pure producer-side fix; no UI, no provider, no service-isolate boundary changes.

</objective>
