## Interaction Contract

### Interrupted Trip Recovery Prompt
- **Entry:** Presented automatically by `DashboardScreen` or `MainShell` if `TrackingNotifier` enters the `interrupted(snapshot)` state on app launch.
- **Position:** Standard centered `AlertDialog`.
- **Dismissible:** Must NOT be dismissible by tapping outside (`barrierDismissible: false`). The user MUST explicitly choose to resume or discard to resolve the interrupted state.
- **Resume Action:** Tapping "Resume" calls `TrackingNotifier.resumeInterruptedTrip()`, dialog closes, screen transitions to active tracking hero.
- **Discard Action:** Tapping "Discard" calls `TrackingNotifier.discardInterruptedTrip()`, dialog closes, file is deleted, state returns to `idle`.
