# Phase 22: Home-Screen Widget Implementation

## Goal
Users can add an Android home-screen widget that starts or stops a commute with one tap and always reflects the current tracking state, including live stats (distance and duration).

## Requirements
- WIDGET-01

<tasks>

<task id="T01">
  <title>Configure Android XML Layout for Widget</title>
  <read_first>
    - android/app/src/main/res/layout/widget_layout.xml
  </read_first>
  <action>
    Create or verify `widget_layout.xml`. It must contain:
    - A main button to toggle start/stop.
    - A visibility-controlled `LinearLayout` containing two `TextView`s for `widget_distance` and `widget_duration`.
    These views allow the widget to display live stats when tracking is active.
  </action>
  <acceptance_criteria>
    - `widget_layout.xml` contains a TextView for distance.
    - `widget_layout.xml` contains a TextView for duration.
  </acceptance_criteria>
</task>

<task id="T02">
  <title>Implement CommuteWidgetProvider (Kotlin)</title>
  <read_first>
    - android/app/src/main/kotlin/traevy/traevy/CommuteWidgetProvider.kt
  </read_first>
  <action>
    Ensure `CommuteWidgetProvider.kt` implements `HomeWidgetProvider`. It should:
    - Read `widget_show_stats`, `widget_distance`, and `widget_duration` from `SharedPreferences`.
    - Update the `RemoteViews` with these values.
    - Toggle visibility of the stats block based on the boolean `widget_show_stats`.
  </action>
  <acceptance_criteria>
    - `CommuteWidgetProvider.kt` exists and overrides `onUpdate`.
    - Code references `widget_distance` and `widget_duration`.
  </acceptance_criteria>
</task>

<task id="T03">
  <title>Sync Stats from TrackingNotifier (Dart)</title>
  <read_first>
    - lib/features/tracking/providers/tracking_providers.dart
  </read_first>
  <action>
    Ensure `TrackingNotifier` periodically saves widget data.
    - When tracking is active, it must call `HomeWidget.saveWidgetData` for `widget_show_stats` (true), `widget_distance`, and `widget_duration`.
    - Use `HomeWidget.updateWidget` to trigger a refresh.
    - Implement a throttle (e.g., 15 seconds) so it doesn't overwhelm the Android UI thread.
  </action>
  <acceptance_criteria>
    - `tracking_providers.dart` contains `HomeWidget.saveWidgetData`.
    - `tracking_providers.dart` contains logic to update `widget_show_stats` based on `TrackingActive`.
  </acceptance_criteria>
</task>

<task id="T04">
  <title>Handle Widget Tap Background Action (Dart)</title>
  <read_first>
    - lib/main.dart
  </read_first>
  <action>
    Ensure the `backgroundCallback` in `main.dart`:
    - Instantiates a Riverpod `ProviderContainer`.
    - Retrieves the `trackingServiceControllerProvider`.
    - Calls `start()` or `stop()` based on the URI passed from the widget click.
    - Disposes of the `ProviderContainer` appropriately.
  </action>
  <acceptance_criteria>
    - `main.dart` contains `backgroundCallback`.
    - `backgroundCallback` creates a `ProviderContainer`.
    - `main.dart` imports `kStopTrackingEvent`.
  </acceptance_criteria>
</task>

</tasks>
