**Settings screen body layout (CONTEXT D-02):**

**Reminder row visibility (CONTEXT D-10, Claude's discretion, resolved):**

- Both rows remain in the widget tree (no `Visibility` or `if` removal) so the layout does not shift height when toggling — height stays consistent regardless of state.
- Opacity 0.38 matches Material 3's disabled-state specification.

## Typography

Inherits from `Theme.of(context).textTheme`. Phase 7 declares **exactly 3 sizes / 2 weights** — the Settings screen is information-dense but has no display numbers. Must not introduce custom `TextStyle` instances except where the table allows.

| Role | TextTheme token | Size (sp) | Weight | Line height | Phase 7 usage |
|------|----------------|-----------|--------|-------------|---------------|
| Section heading | `titleSmall` | 14sp | w500 (medium, theme-provided) | 1.43 | "Appearance" and "Notifications" section header labels; rendered in `colorScheme.primary` |
| Tile title | `bodyLarge` | 16sp | w400 (regular) | 1.50 | `RadioListTile.title`, `SwitchListTile.title`, time `ListTile.title` — all use the Material 3 `ListTile` default which is `bodyLarge` |
| Tile subtitle | `bodyMedium` | 14sp | w400 (regular) | 1.43 | Time `ListTile.subtitle` showing the selected time (e.g., "8:00 AM"); `RadioListTile` does not use subtitles |

**Weight policy:** only **w400 (regular)** is authored explicitly. `titleSmall` w500 is accepted from the theme default and is never authored via `copyWith(fontWeight: ...)`. Never declare custom weights for `ListTile` titles — use Material 3 defaults.

**Time display (CONTEXT Claude's discretion, resolved):**
- Format the `reminderTime` (HH:mm string) for display using `DateFormat.jm().format(DateFormat('HH:mm').parse(reminderTime))`.
- Example output: "8:00 AM" (en_US locale). Use the `intl` package already in `pubspec.yaml`.
- Displayed as `ListTile(subtitle: Text(formattedTime))`.

## Interaction Contract

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Open Settings | Tap gear `IconButton` in Dashboard AppBar | `Navigator.pushNamed(context, kRouteSettings)` |
| Back from Settings | Hardware/gesture back or system back navigation | `Navigator.pop` — returns to Dashboard |
| Toggle dark mode | Tap a `RadioListTile` row ("System default" / "Light" / "Dark") | `dao.upsert(prefs.copyWith(darkMode: value))` → `userPreferenceProvider` emits → `TraevyApp.themeMode` updates instantly (no restart) |
| Toggle weekly summary | Tap `SwitchListTile` for "Weekly summary" | `dao.upsert(prefs.copyWith(weeklyNotificationEnabled: value))`; if `true` → `NotificationService.scheduleWeeklySummary()`; if `false` → `NotificationService.cancelWeeklySummary()` |
| Toggle daily reminder | Tap `SwitchListTile` for "Daily reminder" | `dao.upsert(prefs.copyWith(reminderEnabled: value))`; if `true` → `NotificationService.scheduleReminder()`; if `false` → `NotificationService.cancelReminder()`. Time + weekend rows transition from `opacity: 0.38` to `opacity: 1.0` (200ms) |
| Change reminder time | Tap time `ListTile` (only when `reminderEnabled == true`) | `showTimePicker(context: context, initialTime: ...)` → on confirm: `dao.upsert(prefs.copyWith(reminderTime: hhMm))` → reschedule reminder with new time |
| Toggle weekend reminder | Tap weekend `SwitchListTile` (only when `reminderEnabled == true`) | `dao.upsert(prefs.copyWith(weekendReminder: value))` → cancel + reschedule reminder with updated `DateTimeComponents` strategy |
| Scroll | Vertical drag | Standard Material scroll via `SingleChildScrollView`; no parallax, no sticky headers |

**Theme change immediacy (CONTEXT D-04):**
- `TraevyApp.build()` calls `ref.watch(userPreferenceProvider)`. When the user taps a `RadioListTile`, the DAO upsert triggers a Drift stream emission, which propagates through `userPreferenceProvider`, which causes `TraevyApp` to rebuild with the new `ThemeMode`. The entire app (all routes, all cards, all text) transitions in the same frame. No animation, no restart.
- Fallback during `loading` and `error` states: `ThemeMode.system` (safe default).

**Reminder row enabled/disabled behavior (CONTEXT D-10):**
- When `reminderEnabled == false`: time `ListTile` and weekend `SwitchListTile` render with `enabled: false` AND `AnimatedOpacity(opacity: 0.38)`. They remain in the widget tree.
- When `reminderEnabled == true`: both rows are `enabled: true` AND `AnimatedOpacity(opacity: 1.0)`.
- The `ListTile` with `enabled: false` does NOT fire `onTap` — this is Flutter's built-in behavior, no manual guard needed.

**Time picker result handling:**
- `showTimePicker` returns a `TimeOfDay?`. On null (dismiss), no action is taken.
- Format to HH:mm: `'${time.hour.toString().padLeft(2, '0')}:${time.minute.toString().padLeft(2, '0')}'`.
- Trigger reschedule only if the new time differs from the stored time.

**Settings persistence model:**
- All settings writes are Drift upserts — fully offline, no network involvement.
- Notification scheduling/cancellation occurs synchronously after the upsert in the same event (fire-and-forget — scheduling is async but not awaited in the UI layer, consistent with CLAUDE.md "never block UI on network/async" principle).

## Accessibility

| Rule | Implementation |
|------|----------------|
| Touch target ≥ 44dp | `RadioListTile`, `SwitchListTile`, `ListTile` default heights are 56dp — no override needed |
| Color is not the only signal | `RadioListTile` communicates selection via radio button shape + fill + label text. `SwitchListTile` communicates state via switch position + label text. Dark mode selection is communicated by the checked radio, not color alone. |
| Screen reader labels | `RadioListTile.title` and `SwitchListTile.title` provide accessible labels automatically. Time `ListTile` must include `Semantics(label: 'Reminder time, currently $formattedTime, tap to change')` wrapping the tile when reminder is enabled. |
| Contrast | Inherits Material 3 default scheme — WCAG AA in light, AAA in dark. `colorScheme.primary` section headers meet contrast against `surface` background. |
| Disabled row labeling | When `reminderEnabled == false`, time `ListTile` has `enabled: false` — Flutter's `ListTile` applies `colorScheme.onSurface.withOpacity(0.38)` to content automatically; `AnimatedOpacity` wrapper reinforces this visually. Screen readers should NOT announce disabled rows as interactive. |
| Text scaling | All `ListTile` content uses Material 3 `TextTheme` tokens — never `TextStyle(fontSize: ...)` directly. Rows must reflow height naturally when `textScaler` ≥ 1.5. |

## Claude's Discretion (resolved)

CONTEXT.md flagged these as Claude's discretion. Resolved here so the planner has a fixed contract:

1. **Settings screen AppBar title:** `kSettingsAppBarTitle = 'Settings'` — single word, consistent with `'Stats'` and `'History'` patterns.

2. **Section header styling:** `titleSmall` + `colorScheme.primary` color in a `Padding(EdgeInsets.fromLTRB(16, 16, 16, 8))`. This matches Material 3 settings-page convention (color-accented category labels) without an `ExpansionTile` or custom widget.

3. **Whether weekend toggle uses `enabled:` or `Visibility`:** Use `enabled: reminderEnabled` + `AnimatedOpacity`. Both rows stay in the DOM — no layout jump on toggle. Visual feedback is smooth via opacity animation.

4. **Time picker result display format:** `DateFormat.jm().format(DateFormat('HH:mm').parse(reminderTime))` — displays "8:00 AM" style. Uses `intl` package already in `pubspec.yaml`.

5. **File naming within `lib/features/settings/`:** `screens/settings_screen.dart`, `providers/settings_providers.dart`. `_AppearanceSection` and `_NotificationsSection` are private widgets within `settings_screen.dart` (not separate files — they are implementation details).

6. **Provider naming:** `userPreferenceProvider` (matches RESEARCH.md Pattern 2 exactly). Declared as `final StreamProvider<UserPreferencesValue> userPreferenceProvider = ...` in `lib/features/settings/providers/settings_providers.dart`.

7. **Notification service architecture:** Create `lib/notifications/notification_service.dart` as a new standalone class (`NotificationService`). Do not extend or modify `TrackingNotificationService` (CONTEXT D-14 contract: separate channels, separate class, separate concerns).

8. **Exact empty-week text:** `kWeeklySummaryNotificationBodyEmpty = 'No commutes recorded this week'` — factual, no emoji, no exclamation, consistent with the app's neutral informational tone.

9. **Notification ID allocation (RESEARCH.md Pitfall 6):** `kWeeklySummaryNotificationId = 10`; `kReminderNotificationId = 20`. Reminder weekday-only mode uses IDs 20–24 (Mon=20, Tue=21, Wed=22, Thu=23, Fri=24). Cancel the full range (20–24) before rescheduling.

10. **Channel IDs:** `kWeeklySummaryChannelId = 'weekly_summary'`; `kReminderChannelId = 'commute_reminder'`. Both are distinct from `kTrackingNotificationChannelId` (the existing tracking channel) per D-14.

The implemented screen passes when:

- [ ] **UX-02 entry point:** Gear `IconButton` appears as the 4th (rightmost) trailing icon in the Dashboard AppBar; tapping navigates to `SettingsScreen`.
- [ ] **UX-02 sections:** `SettingsScreen` renders exactly two named sections — "Appearance" (header in `colorScheme.primary`) and "Notifications" — separated by a full-width `Divider`.
- [ ] **UX-02 radio rows:** Three `RadioListTile` rows in the Appearance section ("System default", "Light", "Dark"); the row matching the persisted `darkMode` value is selected on load.
- [ ] **UX-02 instant theme:** Tapping a `RadioListTile` row changes the app's theme (all screens) instantly with no restart; `kDarkModeLight` / `kDarkModeDark` / `kDarkModeSystem` constants used as group values.
- [ ] **UX-04 weekly toggle:** "Weekly summary" `SwitchListTile` reflects `weeklyNotificationEnabled` from Drift; toggling on schedules the Sunday 6pm `zonedSchedule`; toggling off cancels `kWeeklySummaryNotificationId`.
- [ ] **UX-05 reminder toggle:** "Daily reminder" `SwitchListTile` reflects `reminderEnabled`; toggling enables/disables reminder rows with `AnimatedOpacity` transition (200ms).
- [ ] **UX-05 time picker:** Tapping the time `ListTile` opens Flutter's `showTimePicker`; confirming updates `reminderTime` in Drift and reschedules the notification.
- [ ] **UX-05 weekend toggle:** "Include weekends" switch controls `weekendReminder`; false schedules 5 per-weekday `DateTimeComponents.dayOfWeekAndTime` notifications (IDs 20–24); true schedules a single `DateTimeComponents.time` notification (ID 20).
- [ ] **App startup:** Notification channels registered and enabled notifications rescheduled in `main.dart` before `runApp`.
- [ ] **a11y:** No color-only signalling. Time `ListTile` has `Semantics` label when enabled. All interactive rows have tooltip or title.
- [ ] **Constants:** Every visible string, channel ID, notification ID, and timing value lives in `lib/config/constants.dart`. No inline string literals in widget files.
- [ ] **Size:** `SettingsScreen`, `_AppearanceSection`, and `_NotificationsSection` are each under 100 lines. Each public member has a `///` doc comment.
- [ ] **Provider:** No `@riverpod` annotation. `userPreferenceProvider` is a manual `StreamProvider<UserPreferencesValue>`.
- [ ] **Migration:** Schema version bumped to 2; `weeklyNotificationEnabled` column added via `m.addColumn`; `UserPreferencesValue.defaults()` returns `weeklyNotificationEnabled: false`.
