<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Constants → widget layer | String literals used as radio group values; mismatched constants cause silent wrong-group selection |
| Drift migration → SQLite | addColumn operates on real user data; migration must be idempotent and not drop existing data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | database.dart migration | mitigate | Use `if (from < 2)` guard — idempotent; re-running on a v2 DB is a no-op; only `m.addColumn` is called, no DROP or ALTER |
| T-07-01-02 | Information Disclosure | drift_schema_v2.json | accept | Schema snapshot contains no PII; it mirrors the DDL only; committed to version control for migration test reproducibility |
| T-07-01-03 | Denial of Service | USE_EXACT_ALARM manifest | accept | Play Store may audit `USE_EXACT_ALARM` usage; if rejected, fallback is `SCHEDULE_EXACT_ALARM` + runtime permission request; risk is low for a utility app |
| T-07-01-04 | Spoofing | kDarkModeLight/kDarkModeDark constants | mitigate | Constants defined once in constants.dart; all consumers import from that single source; no inline string literals allowed (very_good_analysis enforces no magic strings) |
</threat_model>
