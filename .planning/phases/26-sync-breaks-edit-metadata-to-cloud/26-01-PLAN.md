## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client â†’ `POST /trips/sync` | Untrusted request body crosses here; the 4 new fields are NEW attack surface on this boundary |
| Firestore doc â†’ `GET /trips/restore` response | A doc could in principle be hand-edited in the console (out of client control); the converter must not crash or leak on malformed data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Denial of Service | `tripSchema.breaks` | mitigate | `.max(kMaxBreaksPerTrip)` (50) rejects an oversized breaks array with a 400 BEFORE any Firestore work, mirroring the existing `kMaxRoutePolylineChars`/`kMaxSyncBatchTrips` bound pattern |
| T-26-02 | Tampering | `tripSchema.directionSource` | mitigate | `z.enum(['manual','geofence','time'])` rejects any value outside the closed set, same posture as the existing `direction` enum |
| T-26-03 | Tampering / DoS (poison payload) | `tripSchema.breaks[].startTime/endTime` | mitigate | `z.string().datetime()` per break entry (same validator already used for the trip-level `startTime`/`endTime`) rejects malformed timestamps before they reach Firestore or a downstream restore parse |
| T-26-04 | Information Disclosure | `restore-trips.ts` Trip projection | accept (unchanged) | The projection still strips `deleted`/`deletedAt`/`serverUpdatedAt`; the 4 new fields are ordinary trip data already visible to the owning user, no new disclosure surface |
| T-26-05 | Tampering | Client spoofing `isEdited:true` | accept | `isEdited` is a display/UX flag only (drives an "~estimated" hint), not used for any trust/authorization decision now or foreseeably â€” no mitigation needed |
| T-26-06 | Spoofing / Elevation of Privilege | Auth + ownership (`userId`) on both handlers | accept (unchanged, out of scope) | V2/V4 controls (`verifyAuth`, forced `userId`) are untouched by this plan â€” this phase only extends V5 input validation |
