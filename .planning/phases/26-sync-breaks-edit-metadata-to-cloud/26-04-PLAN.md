<objective>
Implement the one-time backfill (CONTEXT.md D-01/D-02/D-03): find every local trip whose metadata
(breaks, edited flag, direction source) is non-default, re-enqueue it for upload exactly once per
install, guarded by the Plan 02 version-keyed marker, triggered off the SAME auth-transition seam
Phase 24's auto-restore already uses.

Purpose: SC4 ("a one-time backfill re-enqueues local trips that have breaks or edits so their cloud
copies gain the metadata") — without this, a trip created/edited BEFORE this phase shipped would
never get its breaks/edit-flag/direction-source pushed to Firestore, since the sync queue only fires
on create/update, not retroactively.

Output: a new `TripsDao` query identifying backfill candidates, and `MainShell` wired to run the
backfill (enqueue + mark-done) once, sequenced after auto-restore, on every `AuthSignedIn` transition.
</objective>

Future<List<TripRow>> geofenceBackfillCandidates() {
  return (select(trips)
        ..where((t) => t.directionSource.equals(kDirectionSourceManual).not() &
                       t.isManualEntry.equals(false) &
                       t.routePolyline.isNotNull() &
                       t.routePolyline.length.isBiggerThanValue(0))
        ..orderBy([(t) => OrderingTerm(expression: t.startTime, mode: OrderingMode.asc)]))

}
