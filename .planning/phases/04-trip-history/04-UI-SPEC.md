> Registry safety: not applicable — Flutter project, no shadcn/npm registry.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3 via ThemeData) |
| Preset | not applicable |
| Component library | Flutter Material 3 (`material.dart`) |
| Icon library | Material Icons (`Icons.*`) — already used in Phases 1–3 |

**Note:** `lib/config/theme.dart` declares bare `ThemeData.light(useMaterial3: true)` and `ThemeData.dark(useMaterial3: true)` — no custom color seeds or tokens yet (Phase 7 adds branding). Phase 4 relies entirely on Material 3 colorScheme roles resolved at runtime.

## Spacing Scale

Declared values — all multiples of 4. Canonical names match the private constants already established in `edit_trip_sheet.dart` and must be replicated in Phase 4 widget files using the same `_k*` convention.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon-to-label gap, inline badge padding, error text top margin |
| sm | 8dp | Between adjacent row items in a trip card (icon + text gap, button gap in sheets) |
| md | 16dp | Default horizontal screen padding, field-to-field gap, card internal padding |
| lg | 24dp | Section-to-section gap (e.g. top gap below drag handle in sheets, map-to-stats gap) |
| xl | 32dp | Not used in Phase 4 |
| 2xl | 48dp | Not used in Phase 4 |
| 3xl | 64dp | Not used in Phase 4 |

**Exceptions:**
- Map widget height: 256dp (not on 8-point scale — fixed height required by flutter_map, falls between 2xl and 3xl). This is the only allowed exception.
- Touch targets for icon buttons (edit, delete trailing icons): minimum 48dp tap area via `IconButton` default — Android Material 3 spec requirement.
- `SliverStickyHeader` / date-section header row: 40dp tall (not on scale — minimum legible header for a compact list grouping). This is the only second exception.

## Typography

All roles resolved from `Theme.of(context).textTheme.*` — no hardcoded font sizes. The table maps Material 3 textTheme roles to their default values for documentation and implementation guidance.

| Role | TextTheme Role | Default Size | Weight | Line Height | Usage |
|------|---------------|--------------|--------|-------------|-------|
| Screen title | `titleLarge` | 22sp | Medium (500) | 1.27 | AppBar title ("History"), detail screen title (direction label) |
| Section header | `titleMedium` | 16sp | Medium (500) | 1.5 | Sticky date headers ("Today", "Yesterday", "Mon 21 Apr") |
| Body / card primary | `bodyLarge` | 16sp | Regular (400) | 1.5 | Trip departure time in list card, trip duration in detail stats |
| Body / card secondary | `bodyMedium` | 14sp | Regular (400) | 1.43 | Distance in detail stats, "Manually entered" label |
| Label | `labelLarge` | 14sp | Medium (500) | 1.43 | Stats row labels ("Duration", "Distance", "Traffic"), direction chip text |
| Caption / supporting | `bodySmall` | 12sp | Regular (400) | 1.33 | Traffic breakdown sub-labels ("Moving", "Stuck"), calendar event marker count |

**Constraint:** Never hard-code font sizes as numeric literals. Always use `textTheme.*` accessors. Phase 7 may reseed the color scheme but textTheme roles will remain stable.

## Color

All values resolved from `Theme.of(context).colorScheme.*` — no hex literals in Phase 4 code. The table maps Material 3 colorScheme roles to their semantic function.

| Role | colorScheme Role | 60/30/10 | Usage |
|------|-----------------|----------|-------|
| Dominant surface | `colorScheme.surface` | 60% | Screen background (Scaffold backgroundColor), list view background |
| Secondary surface | `colorScheme.surfaceContainerLow` | 30% | Trip cards, stats section background on detail screen |
| Accent | `colorScheme.primary` | 10% | See reserved list below |
| Destructive | `colorScheme.error` | per action | Delete confirmation button background (`FilledButton.styleFrom(backgroundColor: colorScheme.error)`) — pattern from Phase 3 home_screen.dart, replicated identically |

**Accent (`colorScheme.primary`) reserved for:**
1. The direction chip/badge on each trip card (filled chip background when direction is known)
2. The route polyline drawn on the flutter_map tile layer
3. Event marker dots on the table_calendar widget (days that have trips)
4. The active/selected segment of the SegmentedButton used for calendar/list toggle
5. The "View history" TextButton label on the home screen (D-02)

**Accent is NOT used for:** card backgrounds, stats section backgrounds, icon colors in the stats row (use `colorScheme.onSurfaceVariant`), date section headers.

**Section header background:** `colorScheme.surfaceContainerHighest` — provides visual separation between sticky date headers and trip cards without requiring a custom color.

**"Manually entered" label:** `colorScheme.onSurfaceVariant` text on `colorScheme.surfaceContainerLow` background — subdued, not accent-colored, to signal absence of GPS data without alarming the user.

## Screen-by-Screen Interaction Contract

### 1. History List Screen (`lib/features/trips/screens/history_screen.dart`)

**Primary visual anchor:** The departure time (`bodyLarge`) on each trip card — the first thing a user scans when reviewing their history. All other card elements (direction icon, duration, chip) are visually subordinate to it.

**Layout:**
- `Scaffold` with `AppBar(title: Text('History'))`.
- AppBar trailing action: toggle icon button switching between list and calendar views. Icon: `Icons.calendar_month_outlined` when in list view (tapping opens calendar), `Icons.list_rounded` when in calendar view (tapping returns to list). No SegmentedButton in AppBar — icon is sufficient.
- Body: `CustomScrollView` with `SliverList` children grouped by date. Each date group: one sticky header sliver + one sliver of trip cards.

**Sticky date headers:**
- Format rules (from D-03): date == today → "Today"; date == yesterday → "Yesterday"; older → `DateFormat('EEE d MMM')` (e.g. "Mon 21 Apr") using `intl` package already in pubspec.
- Header widget: `Container` with `colorScheme.surfaceContainerHighest` background, 40dp tall, `md` (16dp) horizontal padding, `titleMedium` text.
- Sticky behavior: use `SliverPersistentHeader` with `pinned: true` OR the `sliver_sticky_headers` approach — the planner may choose the simplest Flutter-native implementation. `SliverPersistentHeader` requires no new dependency; prefer it.

**Trip card:**
- `Card` with `colorScheme.surfaceContainerLow` background (Material 3 `Card.filled` variant), `md` (16dp) internal padding.
- Content row (left-to-right):
  - Leading: direction icon (`Icons.arrow_upward_rounded` for to_office, `Icons.arrow_downward_rounded` for to_home, `Icons.help_outline_rounded` for unknown), `colorScheme.onSurfaceVariant` color, 24dp size.
  - Title: departure time formatted as `DateFormat.jm()` (e.g. "8:32 AM"), `bodyLarge` style.
  - Subtitle: duration formatted as "42 min" (under 60 min) or "1h 12min" (60 min and over), `bodyMedium` style, `colorScheme.onSurfaceVariant`.
  - Trailing: direction chip + action icon. Direction chip: `Chip(label: Text('To office'))` or `Chip(label: Text('To home'))`, 12sp label, compact height. Action: `IconButton(icon: Icon(Icons.more_vert), tooltip: 'Trip options')` — tapping opens a `ModalBottomSheet` with two `ListTile` rows: "Edit trip" and "Delete trip". This avoids swipe-to-reveal (complex gesture conflict with scroll) and long-press (not discoverable on Android). The "more" icon is the standard Android overflow pattern.

**Trip options sheet (triggered by `Icons.more_vert`):**
- `showModalBottomSheet` with `showDragHandle: true` — consistent with EditTripSheet pattern.
- Two `ListTile` rows:
  1. `leading: Icon(Icons.edit_outlined)`, `title: Text('Edit trip')` → calls `showModalBottomSheet` with `EditTripSheet(summary: summary)`.
  2. `leading: Icon(Icons.delete_outlined, color: colorScheme.error)`, `title: Text('Delete trip', style: TextStyle(color: colorScheme.error))` → calls `handleDeleteTrip(context, ref, tripId)` (Phase 3 pattern, public method).
- No save/cancel buttons — each option is a direct tap action.

**Tap on trip card body** → `Navigator.pushNamed(context, kRouteTripDetail, arguments: tripId)`.

**Empty state:**
- Centered column: `Icon(Icons.route_outlined, size: 64, color: colorScheme.onSurfaceVariant)`, `SizedBox(height: lg)`, `Text('No trips yet', style: textTheme.titleMedium)`, `SizedBox(height: sm)`, `Text('Start a commute to see your history here.', style: textTheme.bodyMedium, textAlign: TextAlign.center)`.

### 2. Calendar View (shown inside History Screen when toggled)

**Layout:**
- Replaces the `CustomScrollView` list with a `Column`: `TableCalendar` on top (takes its natural height — approximately 350dp), then a `Divider`, then an `Expanded` `ListView` showing only trips for the selected date.

**TableCalendar configuration:**
- `focusedDay`: initially today, updates when user taps a date.
- `selectedDayPredicate`: highlights the currently tapped day.
- `eventLoader`: returns a `List<TripSummary>` for each day — the list screen already holds all trips from `watchAllSummaries()`, so eventLoader performs a client-side lookup. No new provider needed.
- `calendarFormat`: `CalendarFormat.month` fixed — no week/2-week toggle (YAGNI).
- `headerStyle`: `formatButtonVisible: false` (no format toggle button).
- `calendarStyle`:
  - `markerDecoration`: `BoxDecoration(color: colorScheme.primary, shape: BoxShape.circle)` — accent dot under days with trips.
  - `selectedDecoration`: `BoxDecoration(color: colorScheme.primary, shape: BoxShape.circle)`.
  - `todayDecoration`: `BoxDecoration(color: colorScheme.primaryContainer, shape: BoxShape.circle)`.
  - `markersMaxCount`: 1 — one dot per day regardless of trip count (simplicity).
- `firstDay`: `DateTime.utc(2020, 1, 1)` (covers historical imports).
- `lastDay`: `DateTime.now()` (no future dates).

**Filtered list below calendar:**
- When a date is selected, show a plain `ListView` of trip cards (same card widget as history list, no date headers).
- When no trips exist for selected date: `Text('No trips on this day.', style: textTheme.bodyMedium)` centered in remaining space.
- When no date is selected (initial state): show today's trips or `Text('Tap a date to see trips.', style: textTheme.bodyMedium)` if no trips today.

### 3. Trip Detail Screen (`lib/features/trips/screens/trip_detail_screen.dart`)

**Routing:** named route `kRouteTripDetail = '/trip-detail'`; `tripId` passed as `ModalRoute.of(context)!.settings.arguments as String`.

**Data fetch:** `TripsDao.findById(tripId)` called once in `initState` (D-07). Screen has two states: loading and loaded. Loading state: `Center(child: CircularProgressIndicator())`. Loaded state: the full layout below.

**GPS trip layout (D-06):**

```
Scaffold
  AppBar(title: Text(direction label), actions: [edit icon, delete icon])
  body: CustomScrollView
    SliverToBoxAdapter  ← map widget, 256dp tall
    SliverToBoxAdapter  ← stats section
```

- **AppBar actions (GPS trips):**
  - `IconButton(icon: Icon(Icons.edit_outlined), tooltip: 'Edit trip')` → opens `EditTripSheet`.
  - `IconButton(icon: Icon(Icons.delete_outline), tooltip: 'Delete trip', color: colorScheme.error)` → calls `handleDeleteTrip`. After successful delete, `Navigator.of(context).pop()` returns to history list.

- **Map widget (GPS trips, D-01):**
  - `SizedBox(height: 256, child: FlutterMap(...))`.
  - `TileLayer` using OpenStreetMap: `urlTemplate: 'https://tile.openstreetmap.org/{z}/{x}/{y}.png'`.
  - `PolylineLayer` with a single `Polyline`: `points` from `decodePolyline(trip.routePolyline)` (existing util), `color: colorScheme.primary`, `strokeWidth: 4.0`.
  - `MapController` with `initialCameraFit` set to `CameraFit.bounds(bounds: LatLngBounds.fromPoints(points), padding: EdgeInsets.all(32))` — route auto-fits inside the map tile, 32dp padding on each side.
  - No zoom controls, no attribution text visible (OpenStreetMap license attribution is required — add `RichAttributionWidget` with OpenStreetMap credit, collapsed/overlay style to keep map clean).
  - Map is not interactive (user cannot scroll/zoom the detail screen into the map). `interactiveFlags: InteractiveFlag.none` to prevent accidental map pan eating list scroll events.

- **Stats section (GPS trips):**
  - `Padding(padding: EdgeInsets.all(16), child: Column(...))`.
  - Each stat row: `_StatRow(label: '...', value: '...')` — private widget, icon on left (`colorScheme.onSurfaceVariant`, 20dp), label (`labelLarge`), value right-aligned (`bodyLarge`).
  - Stat rows in order:
    1. `Icons.schedule_outlined` · "Duration" · formatted as "42 min" or "1h 12min"
    2. `Icons.straighten_outlined` · "Distance" · formatted as "12.4 km" (meters ÷ 1000, one decimal place)
    3. `Icons.explore_outlined` · "Direction" · "To office" or "To home"
    4. `Icons.calendar_today_outlined` · "Date" · `DateFormat('EEE, d MMM yyyy')` (e.g. "Mon, 21 Apr 2025")
    5. `Icons.timer_outlined` · "Moving" · formatted as "34 min" (time_moving_seconds)
    6. `Icons.traffic_outlined` · "Stuck in traffic" · formatted as "8 min" (time_stuck_seconds)

**Manual trip layout (D-05):**

```
Scaffold
  AppBar(title: Text(direction label), actions: [edit icon, delete icon])
  body: Padding(all: 16)
    Column
      "Manually entered" badge
      SizedBox(height: 24)
      stat rows (duration, direction, date — NO distance, NO moving/stuck)
```

- **"Manually entered" badge:**
  - `Chip(label: Text('Manually entered — no route recorded'), avatar: Icon(Icons.edit_note_outlined, size: 16))`.
  - Uses default chip styling — no custom color. `colorScheme.surfaceContainerLow` background, `bodySmall` label, `colorScheme.onSurfaceVariant` text.
  - Placed at the top of the stats column, full-width or wrapped.

- **Manual trip stats rows (subset):**
  1. `Icons.schedule_outlined` · "Duration" · formatted as "42 min" or "1h 12min"
  2. `Icons.explore_outlined` · "Direction" · "To office" or "To home"
  3. `Icons.calendar_today_outlined` · "Date" · `DateFormat('EEE, d MMM yyyy')`
  - Distance row omitted (no GPS distance).
  - Moving/stuck rows omitted (no GPS speed data).

- **AppBar actions (manual trips):** same edit + delete icons as GPS trips.

- Sentence case throughout (no title case).

- No emoji.

- Duration formatting: `< 60 minutes` → "N min"; `≥ 60 minutes` → "NhNNmin" (e.g. "1h 12min"). No "hours" or "minutes" spelled out. Format defined once as a utility function in `lib/shared/utils/` — not inlined per CLAUDE.md no-hardcoded-strings rule.
- Distance formatting: meters ÷ 1000, one decimal place, " km" suffix. No "kilometers" spelled out.
