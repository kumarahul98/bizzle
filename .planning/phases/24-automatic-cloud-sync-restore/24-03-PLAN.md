<objective>
Implement conflict detection and reconciliation UI for cloud restores.

Purpose: Ensures that when downloading trips from the cloud, conflicting edits or overlapping recordings do not overwrite local data without user consent.
Output: Conflict detection logic in RestoreController and a ConflictResolutionSheet for user decisions.
</objective>

## Artifacts this phase produces
- `ConflictResolutionSheet` widget class in `lib/features/settings/widgets/conflict_resolution_sheet.dart`
- `RestoreConflict` class (and subclasses like `SameUuidConflict`, `OverlapConflict`) in `lib/sync/restore_controller.dart` or related models file
- Modified `restore()` method in `RestoreController` to emit `RestoreState.conflict`
- Extended listener logic in `MainShell` to present the `ConflictResolutionSheet` when `RestoreState.conflict` is emitted
