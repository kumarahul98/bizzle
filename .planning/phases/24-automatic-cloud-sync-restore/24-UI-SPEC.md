## Interaction Contract

### Auto-restore (sign-in trigger)
- **Entry:** fires automatically on `AuthSignedIn` transition — user sees no modal, no tap required
- **Progress:** `SnackBar` with `kAutoRestoreInProgress` appears immediately; no spinner blocks UI
- **Success:** `SnackBar` updates to result copy (`kAutoRestoreResultTemplate` / `kAutoRestoreUpToDate`)
- **Error:** `SnackBar` shows `kAutoRestoreError` with a text link to Settings
- **Duration:** SnackBar auto-dismisses after 4 s (Flutter default); no persistent UI after success

### Stuck-item banner (dashboard)
- **Entry:** `MaterialBanner` at top of dashboard body when `SyncFailed` and auto-retry window exhausted
- **Position:** above the trip list, below the app bar — same layer as `PermissionBanner` in tracking screen
- **Action:** "Review in Settings" navigates to Settings → Account section
- **Dismiss:** "Dismiss" button removes the banner for the session (no persistence needed; re-appears on next launch if still stuck)
- **Never interrupts:** background-only — the FAB and trip list remain fully interactive

### CloudSyncRow (Settings) — extended subtitle states
Extend existing `CloudSyncRow` subtitle `switch` with auto-retry states:

| `SyncStatus` | Subtitle |
|---|---|
| `SyncIdle` / `SyncSynced` | `'All synced'` |
| `SyncSyncing` | `'Syncing…'` |
| `SyncOffline` (pending > 0) | `'N pending'` |
| `SyncOffline` (pending = 0) | `'Offline'` |
| `SyncFailed` | `'Sync failed — tap to retry'` *(unchanged)* |

No new `SyncStatus` variants needed for the auto-retry time-gate (it is internal state in the engine, not surfaced in the status sealed class).
