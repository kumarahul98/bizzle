<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live Activity surface → app (URL scheme) | A tap on the Stop button crosses from the OS-owned ActivityKit surface back into the app via the traevy:// URL scheme |
| App Group shared container → Widget Extension | The extension reads ContentState fields from the shared UserDefaults suite |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-08 | Spoofing | traevy:// URL scheme | accept | iOS routes a custom URL scheme only to the registered app; another app cannot register the same scheme to hijack it (unlike Android intents). Low risk per RESEARCH Security Domain. The Dart-side listener additionally exact-matches host=='stop' (T-15-09, Plan 05). |
| T-15-09 | Information Disclosure | Live Activity rendered fields | mitigate | The widget renders only pre-formatted display strings from ContentState (elapsedFormatted/distanceFormatted/...); no raw coordinates are ever bridged into the App Group container (V5 input validation, ASVS). |
| T-15-10 | Tampering | App Group UserDefaults suite | accept | Suite is sandboxed to the signed team's app + embedded extension; not writable by other apps. |
</threat_model>
