## Accomplishments

- Added 3 regression tests to the `SyncEngine auto-retry time gate` group, all passing against the pre-rename code first (proving the existing gate behavior is correct):
  - `second trigger within window does not re-fire retryFailed` — a second auto trigger inside `kFailedAutoRetryWindow` leaves the failed row untouched and makes zero new API calls
  - `trigger after window elapses DOES re-fire retryFailed` — advancing the injected clock past the window reopens the gate and drains the failed row to synced
  - `manual retryFailed stamps the window, suppressing a following auto trigger` — pins D-03: manual retry is never blocked and its stamp suppresses the next auto trigger
- Renamed `isAutoRetryExhausted` → `autoRetryWindowElapsed` with an identical truth table (true = window elapsed, safe to auto-retry) and a doc comment stating the non-inverted meaning
- Replaced the two inline duplicated gate conditions (connectivity-restore listener, `handleResume()`) with direct getter calls — the condition now exists in exactly one place; `_StuckBannerGate` updated identifier-only with the `!` negation preserved
- Zero behavior change proven: full suite (602 tests) green before and after the rename; `grep -rn "isAutoRetryExhausted" lib/ test/` returns zero matches
