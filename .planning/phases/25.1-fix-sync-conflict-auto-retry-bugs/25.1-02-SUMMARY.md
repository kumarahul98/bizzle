### Auto-fixed Issues

**1. [Rule 3 - Blocking] Enlarged test viewport so the distanceMeters 'Cloud' segment is hittable**
- **Found during:** Task 2 (new D-08 test)
- **Issue:** At the default 800x600 test surface, the plan's scoped-finder tap on the distanceMeters row's 'Cloud' segment derived an offset that failed hit-testing (the segment sat under the bottom sheet's clip; `warnIfMissed` fired and the selection was silently dropped, so `distanceMeters` stayed 12000.0 and the assertion failed). `ensureVisible` did not resolve it because the sheet's own layout, not scroll position, was the constraint.
- **Fix:** Set `tester.view.physicalSize = const Size(800, 1600)` / `devicePixelRatio = 1.0` with `addTearDown(tester.view.reset)` at the top of the new test only, keeping the plan's exact finder/ensureVisible/tap pattern. All five field rows and the bulk buttons are hittable without scrolling.
- **Files modified:** test/widget/features/settings/conflict_resolution_sheet_test.dart
- **Verification:** Test passes; assertions confirm the tap registered (distanceMeters == 20000.0)
