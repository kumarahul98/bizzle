## Decisions Made

- **iOS 15.0 minimum (user-approved):** Original plan specified iOS 14.0. During `pod install`, firebase_auth ^6.5.1 and firebase_core ^4.9.0 podspecs imposed an iOS 15.0 floor, causing CocoaPods to reject the 14.0 target. The deployment target was raised to 15.0 in both the `platform :ios` line and the post_install hook. The user explicitly approved this change; PLAN.md, RESEARCH.md, and VALIDATION.md docs were updated (commit 1e4b6f6).
- **flutter precache --ios prerequisite:** `pod install` failed on the first attempt because Flutter.xcframework was not in the local cache. Running `flutter precache --ios` populated the cache and unblocked pod install. This is a one-time machine-setup step.
- **SPM absent:** flutter create with Flutter 3.41.6 + Xcode 26.5 did NOT generate ios/Package.swift. The project is CocoaPods-only. No SPM/CocoaPods conflict to resolve.
- **AppDelegate unchanged:** google_sign_in_ios v7 handles the OAuth redirect via GeneratedPluginRegistrant. No GMSServices call is needed (app uses flutter_map, not google_maps_flutter).

## Deviations from Plan

### User-Approved Changes

**1. [User-Approved] iOS deployment target raised from 14.0 to 15.0**

- **Committed in:** 60b3130 (Task 2 commit), 1e4b6f6 (docs update)

**2. [Rule 3 - Blocking] flutter precache --ios required before pod install**
- **Found during:** Task 2 (pod install)
- **Issue:** Flutter.xcframework was absent from the local CocoaPods cache on this machine. `pod install` failed with a missing framework error because the Flutter engine binary had not been downloaded to the cache directory.
- **Fix:** Ran `flutter precache --ios` to populate the local cache, then re-ran `pod install` successfully.
- **Files modified:** No source files changed — cache-population is a machine-setup step; ios/Podfile.lock reflects the successful pod install
- **Verification:** `pod install` exited 0 after precache; 28 pods resolved
- **Committed in:** 60b3130 (Task 2 commit)
