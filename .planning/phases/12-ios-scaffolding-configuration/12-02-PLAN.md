<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device network stack ↔ remote endpoints | App makes HTTPS calls to CARTO tiles, Google OAuth, and Cloud Functions; ATS governs which transports are permitted |
| app sandbox ↔ iOS Keychain | flutter_secure_storage writes the Firebase ID token; keychain-access-groups entitlement scopes access |
| app bundle ↔ Google Sign-In OAuth redirect | reversed-client-ID URL scheme routes the OAuth callback back into the app |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04 | Tampering | NSAppTransportSecurity | mitigate | Do NOT set NSAllowsArbitraryLoads or any global ATS disable; all endpoints are HTTPS so the default ATS posture (TLS-required) is kept. Only a scoped per-domain exception is permitted if a real ATS error appears, never a global allow. Verified by `grep -c NSAllowsArbitraryLoads` returning 0 |
| T-12-05 | Information disclosure | keychain-access-groups entitlement | mitigate | Use an empty keychain-access-groups array (single-app group). No cross-app sharing group is declared, so the ID token is not exposed to other apps; scope is the narrowest that still satisfies flutter_secure_storage on real devices |
| T-12-06 | Information disclosure | GoogleService-Info.plist committed to repo | accept | GoogleService-Info.plist is client configuration (API keys are client identifiers, not secrets) and is designed to ship inside the app bundle; committing it is the standard FlutterFire workflow and Firestore Security Rules deny-all to clients enforce the real access boundary server-side |
| T-12-07 | Spoofing | reversed-client-ID URL scheme | accept | The OAuth redirect scheme is a public client identifier; Firebase/Google validate the OAuth exchange server-side, so a registered scheme alone grants no privilege. Must match GoogleService-Info.plist REVERSED_CLIENT_ID exactly (verified in Task 3) |
</threat_model>
