<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer Mac ↔ physical iPhone | Locally-signed debug build is installed onto hardware over a trusted USB connection |
| Apple ID free provisioning ↔ device | A free (7-day) provisioning profile authorizes the app to run on this specific device |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-08 | Elevation of privilege | Developer Mode on the iPhone | accept | Developer Mode is the user enabling local install on their own test device for this milestone only; it is the Apple-sanctioned path for free provisioning and is scoped to the developer's own hardware. No production/distribution implication |
| T-12-09 | Spoofing | free provisioning signing identity | accept | A free Apple ID identity self-signs the debug build for the developer's own device; it cannot distribute to other devices and expires in 7 days, so the blast radius is a single trusted device for a short window |
| T-12-10 | Repudiation | DEVELOPMENT_TEAM written to project.pbxproj | accept | The team identifier is a non-secret Apple developer team id; committing it is standard and carries no credential. No mitigation required |
</threat_model>
