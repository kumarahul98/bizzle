<objective>
Add the TRACK-12 quick direction selector. During an active trip the user gets a 1-tap segmented To office / To home toggle (in/under the `RecordingHeader`) that overrides the time-of-day auto-label; the override propagates live to the header label, to the foreground notification, and — at finalize — to the persisted `trips.direction`. From the trip detail screen the same 1-tap toggle changes a saved trip's direction through the existing trips DAO update path. No Drift schema change, no migration.

Purpose: Today direction is purely a time-of-day heuristic (`DirectionLabelService`), recomputed in two places — `HeroRecordCard._resolveDirection` for the live label/notification and `persistFinalizedTrip` for the saved row. A user whose commute doesn't fit the cutoff (e.g. a midday trip home) has no way to correct it during tracking, and correcting it after the fact requires the full edit sheet. D-04..D-07 add a manual override that wins over the heuristic both live and at save, plus a quick trip-view toggle, reusing the existing edit/DAO plumbing.
Output: a reusable `DirectionSegmentedToggle`; a `manualDirectionOverride` + `resolvedDirection` getter + `setDirection` on `TrackingNotifier`; an optional `directionOverride` on `persistFinalizedTrip`; the toggle wired into `_HeroActive` and the trip detail screen; and unit + widget tests proving override-wins-at-finalize, resolved-getter-for-notification, and toggle behaviour.
</objective>
