<objective>
Fix the active-recording elapsed timer (`ElapsedDisplay`) so the 76sp HH:MM:SS mono timer ALWAYS renders fully on one line — no wrapping the last digit, no clipping — at any duration (including 2-digit hours) and any system text-scale (UX-06 / ROADMAP SC#1). The format function and the rendered string are unchanged; this is a layout-fitting fix plus a tabular-figures style tweak.

Purpose: At long durations (HH ≥ 10) or large accessibility text-scales, the current plain `Text` at 76sp / letterSpacing -3 overflows its row inside `_HeroActive` (the live surface in `HeroRecordCard`) and the renderer wraps or clips the final digit. D-01 mandates a `FittedBox(BoxFit.scaleDown)` which mathematically guarantees the glyphs shrink to fit rather than wrap/clip; D-02 requires giving that FittedBox a real width constraint to shrink against; D-03 adds `FontFeature.tabularFigures()` so per-digit width stays constant as the timer ticks.
Output: a fitted, jitter-free `ElapsedDisplay`; a `fontFeatures` param on `TraevyFonts.mono`; and a widget regression test proving no overflow at 2-digit hours and at textScaler 2.0.
</objective>

Current ElapsedDisplay (lib/features/tracking/widgets/elapsed_display.dart) — the bug source.
It renders a Column(mainAxisSize.min) with a SectionLabel and a plain Text at:
```dart
Text(
  _formatElapsed(durationSeconds),       // 'HH:MM:SS' — KEEP UNCHANGED
  style: TraevyFonts.mono(
    size: 76, weight: FontWeight.w500, color: onSurface, letterSpacing: -3,
  ),
)
```
`_formatElapsed` always zero-pads to HH:MM:SS. It MUST remain byte-for-byte identical (D-01 regression guard).

Current TraevyFonts.mono (lib/config/theme.dart ~line 223) has NO fontFeatures param:
```dart
static TextStyle mono({
  required double size,
  FontWeight weight = FontWeight.w400,
  Color? color,
  double letterSpacing = 0,
}) => TextStyle(
  fontFamily: kFontMono,
  fontSize: size, fontWeight: weight, color: color, letterSpacing: letterSpacing,
);
```
Add an optional `List<FontFeature>? fontFeatures` param and forward it to the TextStyle. `FontFeature` is in `dart:ui` (already transitively available; import `package:flutter/material.dart` re-exports it — verify the import). Default null keeps every other `mono(...)` call site unchanged.

Consumer surface: `_HeroActive` in lib/features/dashboard/widgets/hero_record_card.dart renders `ElapsedDisplay(durationSeconds: elapsedSeconds)` inside a `Column(crossAxisAlignment.stretch)` within a Card padded 28h/24v. The available width is the card content width — the FittedBox must take that full width (Column already stretches; ElapsedDisplay's inner Column(mainAxisSize.min) does NOT, so wrap the Text in a full-width box per D-02).

Existing widget-test harness pattern: pump with `MaterialApp(theme: buildLightTheme(), home: Scaffold(body: ...))`. For text-scale, wrap in `MediaQuery(data: MediaQueryData(textScaler: TextScaler.linear(2.0)), child: ...)`. Overflow surfaces as a thrown FlutterError during pump (RenderFlex/paragraph overflow) and/or `tester.takeException()`.
