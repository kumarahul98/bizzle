<objective>
Lay the schema foundation for Phase 18 pause/breaks. Add a normalized `trip_breaks` table (1:N to trips), a denormalized `total_paused_seconds` aggregate column on `trips`, and an `auto_pause_enabled` opt-in column on `user_preferences` — all in a single v2→v3 Drift migration that preserves every existing row. Add a `TripBreaksDao` and a v3 schema-migration test mirroring the existing convention. This plan is the foundation every other Phase 18 plan depends on (TRACK-09, TRACK-10).

Purpose: Phase 19 (full trip editing) must edit individual break segments, so breaks are a normalized table — not a JSON column (D-01, Gemini). `total_paused_seconds` is denormalized onto `trips` so the daily-log/list and stats render without a JOIN (D-02). Redefining `trips.duration_seconds` as ACTIVE duration (wall-clock − paused) is safe for all historical rows because with no breaks `total_paused_seconds` is 0, so active == wall-clock and existing rows are unchanged (D-03).
Output: a v3 schema, `trip_breaks` table + DAO, two new columns with safe defaults, and an automated migration + DAO test suite. NO behaviour change to existing tracking/persist yet — that lands in Plan 02.
</objective>
