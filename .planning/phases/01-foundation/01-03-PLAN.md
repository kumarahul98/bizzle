```dart
import 'package:drift/drift.dart';
import 'package:drift_flutter/drift_flutter.dart';
import 'package:path_provider/path_provider.dart';

import 'package:traevy/config/constants.dart';
import 'daos/sync_queue_dao.dart';
import 'daos/trips_dao.dart';
import 'daos/user_preferences_dao.dart';
import 'tables/sync_queue_table.dart';
import 'tables/trips_table.dart';
import 'tables/user_preferences_table.dart';

part 'database.g.dart';

@DriftDatabase(
  tables: [Trips, SyncQueue, UserPreferences],
  daos: [TripsDao, SyncQueueDao, UserPreferencesDao],
)
class AppDatabase extends _$AppDatabase {
  AppDatabase([QueryExecutor? executor])
      : super(executor ?? _openConnection());

  @override
  int get schemaVersion => 1;

  @override
  MigrationStrategy get migration => MigrationStrategy(
        onCreate: (m) async {
          await m.createAll();
          // D-04: Do NOT seed user_preferences.
          // The DAO handles missing rows with defaults.
        },
        onUpgrade: (m, from, to) async {
          // No upgrades yet at schemaVersion 1.
        },
        beforeOpen: (details) async {
          await customStatement('PRAGMA foreign_keys = ON');
        },
      );

  static QueryExecutor _openConnection() {
    return driftDatabase(
      name: kDatabaseName,
      native: const DriftNativeOptions(
        databaseDirectory: getApplicationSupportDirectory,
      ),
    );
  }
}
```

- `getApplicationSupportDirectory` (from `path_provider`) is the correct directory,
  not `getApplicationDocumentsDirectory` — see the anti-patterns list in RESEARCH.md.
- `PRAGMA foreign_keys = ON` is safe to run and lets Drift enforce FK constraints

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App code → SQLite file on disk | DAO writes cross a trust boundary into the on-device SQLite file. The file is in app-support (not shared). |
| DAO API → UI code | DAOs return typed values; UI code trusts them fully. Input validation happens before the DAO call, not inside it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Information Disclosure | SQLite file in app-support directory | mitigate | `getApplicationSupportDirectory` is used (not documents directory). On Android, app-support is inside the app's sandboxed data dir and is not world-readable. On iOS (future) it's also excluded from iCloud backup. |
| T-01-09 | Tampering | Trip data integrity | mitigate | All inserts go through DAO methods that take typed `Companion` objects — no raw SQL from UI code. Drift's generated companions enforce non-null constraints at compile time. |
| T-01-10 | Denial of Service | DB opens on hot path | mitigate | `@Riverpod(keepAlive: true)` ensures the DB opens once per app lifetime, not per widget. `ref.onDispose(db.close)` guarantees clean shutdown on provider invalidation. |
| T-01-11 | Information Disclosure | Polyline column loaded unnecessarily (Pitfall 7) | mitigate | `TripsDao.watchAllSummaries()` projects into `TripSummary` which omits `routePolyline`. Only `TripsDao.findById()` (detail screen use only) returns the full row. |
