## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test code → production DB | DAO tests use in-memory DB only; real on-disk DB is never touched |
| drift_dev CLI → committed schema snapshot | The schema dump command writes files committed to git; stale snapshots are a silent correctness risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-14 | Tampering | Stale schema snapshot | mitigate | Task 2 commits drift_schemas/drift_schema_v1.json from a fresh dump. Future schema changes must re-run the dump; a CI check can enforce this later. The migration scaffold test fails loudly if the snapshot can't be opened. |
| T-01-15 | Information Disclosure | Test fixtures leaking into logs | accept | DAO tests use synthetic UUIDs and placeholder data ('trip-1', 'local_user'). No real user data. |
| T-01-16 | Denial of Service | Leaked timers from missing closeStreamsSynchronously | mitigate | RESEARCH.md Pattern 6 explicitly requires `closeStreamsSynchronously: true` in the DatabaseConnection. Task 1 verification greps for it. |
| T-01-17 | Repudiation | Untested D-13 payload contract | mitigate | sync_queue_dao_test explicitly asserts payload == null for create/update and payload != null for delete. The contract is now executable, not just documented. |
</threat_model>
