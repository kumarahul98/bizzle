Exact content (per D-11 and Pitfall 6):

```yaml
include: package:very_good_analysis/analysis_options.yaml

analyzer:
  exclude:
    - lib/**/*.g.dart
    - lib/**/*.freezed.dart
    - test/generated_migrations/**
    - drift_schemas/**
    - build/**
  language:
    strict-casts: true
    strict-inference: true
    strict-raw-types: true
  plugins:
    - custom_lint
  errors:
    # Internal app — not publishing a public API.
    public_member_api_docs: ignore
    # Drift companion constructors use positional booleans by design.
    avoid_positional_boolean_parameters: warning

linter:
  rules:
    prefer_const_constructors: true
    prefer_const_literals_to_create_immutables: true
    avoid_dynamic_calls: true
```

Rationale (all citations point back to RESEARCH.md Pitfall 6 and Pitfall 8):
- `exclude: lib/**/*.g.dart` — generated Drift/Riverpod code is excluded or every
  `build_runner` re-run produces lint noise (Pitfall 6).
- `exclude: test/generated_migrations/**` — drift_dev's `schema generate` output
  is auto-generated; Phase 1 scaffolds this in plan 04.
- `public_member_api_docs: ignore` — this is an internal app, not a published package.
  Without this, every Drift table and DAO method would require dartdoc comments.
- `plugins: - custom_lint` — wires the riverpod_lint plugin (Pitfall 8). Without this
  line, `riverpod_lint` is installed but never runs.

After writing the file, run `flutter analyze` on the full project. It must return
zero errors and zero warnings on the hand-written code from tasks 1 and 2. If it
reports findings on the default `main.dart` from plan 01, that's fine — task 4
replaces `main.dart`. Re-run `flutter analyze` at the end of task 4 as the final
gate.

Then run `dart run custom_lint` once to confirm the Riverpod lint plugin is
actually loaded (Pitfall 8 warning sign check). It may report findings on the
default scaffold — those are addressed by task 4 overwriting `main.dart`.
