| Google account picker → app | OAuth flow returns a Google idToken; trusted only after Firebase signInWithCredential validates it |
| app → flutter_secure_storage | ID token written to Android Keystore-backed storage |
| Firebase SDK → app | authStateChanges() / currentUser drive auth state; SDK manages refresh token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | Information Disclosure | ID token storage | mitigate | Token written to flutter_secure_storage (Android Keystore) under kFirebaseIdTokenKey; never SharedPreferences/plaintext (CLAUDE.md, D-10). |
| T-09-03-02 | Information Disclosure | token logging | mitigate | No print/log/debugPrint of idToken, credential, or getIdToken() output (Security Domain); enforced by Task 2 acceptance grep. |
| T-09-03-03 | Spoofing | missing serverClientId | mitigate | GoogleSignIn.instance.initialize(serverClientId: kGoogleServerClientId) — Web client ID required on Android for a Firebase-usable idToken (Pitfall 2). |
| T-09-03-04 | Denial of Service (self) | boot crash on missing config | mitigate | try/catch around initializeApp → degrade to guest (D-15, Pitfall 5); no crash on dev/CI builds. |
| T-09-03-05 | Spoofing | stale token reuse | accept | FlutterFire auto-refreshes ID tokens; Phase 11 reads a fresh getIdToken() per request. Deferred — no custom refresh this phase. |
| T-09-03-06 | Tampering | userId backfill race | mitigate | Backfill awaited inside a single transaction before signIn() returns (Pitfall 7); auth stream re-render cannot observe a stale userId. |
