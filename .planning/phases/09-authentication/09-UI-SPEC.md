# Phase 9 ‚Äî UI Design Contract

>

## Typography

All sizes/weights below already exist in Phase 8's `theme.dart` text theme and `TraevyFonts`. Phase 9 reuses them verbatim. **3 sizes carry the bulk of Phase 9 surfaces (15, 16, 36); 2 weights dominate (w500/w600 and w700).**

| Role | Size | Weight | Line Height | Font | Phase 9 usage |
|------|------|--------|-------------|------|---------------|
| Display | 36px | 700 (w700) | 1.05 | Inter | Onboarding headline (existing); confirmation screen "You're signed in." headline |
| Heading | 22px | 700 (w700) | 1.2 (default) | Inter | Sign-in sheet "Back up your commutes" headline |
| Body | 16px | 400 (w400) | 1.5 | Inter | Onboarding subtext (existing); sheet subtext "Your trips sync automatically when you sign in."; confirmation body |
| Account name | 15px | 600 (w600) | default | Inter | `AccountRow` name (existing); "Sign in to back up" row label |
| CTA / button label | 14px | 600 (w600) | default | Inter | "Continue with Google" (existing); "Let's go" confirmation button; "Sign in later" link (w500) |

| Caption / terms | 11px | 400 (w400) | default | Inter | "By continuing you agree to our terms." (existing); "Sign-in not configured" tooltip text |

**Rules:**
- Numeric and email values use **JetBrains Mono** (`TraevyFonts.mono`); all other text uses **Inter** (`TraevyFonts.ui`). This is the Phase 8 split ‚Äî do not deviate.
- Letter-spacing for 36px display is **-1.2**, for 22px heading **-0.6** (negative tracking on large display text, per existing theme).

## Color

The Traevy palette has light + dark variants (18 tokens each) defined in `TraevyTokens`. Phase 9 introduces no new colors. The 60/30/10 split below maps to existing tokens.

| Role | Token (light / dark) | Usage |
|------|---------------------|-------|
| Dominant (60%) | `bg` ‚Äî `#FAFAF7` / `#1A1B22` | Splash background, screen scaffolds, confirmation screen fill |
| Secondary (30%) | `bgElev` / `surface` ‚Äî `#FFFFFF`,`#F5F5F0` / `#22242E`,`#24262F` | CTA button fill (`bgElev`), bottom sheet surface, card/row surfaces |
| Accent (10%) | `accent` ‚Äî `#3A5F8F` / `#8AABCF` (+ `accentBg` tint) | See reserved-for list below |
| Destructive | `danger` ‚Äî `#C0392B` / `#E05A4A` | "Sign out" row label (existing, stays no-op visual this phase). No new destructive actions in Phase 9. |
| Dimmed text | `textDim` ‚Äî `#6B6B7A` / `#A0A0B8` | Subtext, secondary labels, "Sign in later" link, email |
| Muted text | `textMuted` ‚Äî `#9A9AAA` / `#6E6E88` | Terms caption, disabled-button label, "Sign-in not configured" tooltip |
| Strong border | `borderStr` ‚Äî `#D4D4CE` / `#383A4A` | CTA button outline |

**Accent reserved for (explicit ‚Äî never "all interactive elements"):**
1. `AccountRow` avatar circle ‚Äî `accentBg` fill with `accent`-colored initial letter (existing, signed-in state).
2. The "Sign in to back up" guest row's leading Google/account glyph tint and label emphasis (matches the accent-on-`accentBg` avatar treatment so guest‚Üîsigned-in transition is visually coherent).
3. Confirmation screen avatar circle (same `accentBg` + `accent` treatment as `AccountRow`).

**Accent is NOT used for:**
- The "Continue with Google" CTA button ‚Äî it is a neutral `bgElev` surface with `borderStr` outline and `onSurface` text/icon (existing Phase 8 treatment; the Google button is intentionally neutral, not accent-filled).
- The "Let's go" confirmation button ‚Äî neutral surface treatment, consistent with the Google CTA.

**Disabled state (D-15, Firebase unconfigured):**
- Disabled CTA: reduce opacity to **~38%** (Material disabled convention) on the existing `bgElev`/`borderStr`/text treatment; remove `InkWell` ripple (null `onTap`). Render the "Sign-in not configured" tooltip in `textMuted` 11px. Dev/test builds only ‚Äî production builds with valid config never show this.

## Surface Specifications (NEW this phase)

These three surfaces are new and need the most executor detail. Existing scaffolds (onboarding screen, `GoogleContinueButton`, `AccountRow`, settings rows) are reused as-is ‚Äî only their wiring/state changes, not their visuals.

### A. Static Splash (D-04 ‚Äî `AuthState.loading`)
- **Container** filled with `TraevyTokensExt.bg` (full screen, edge-to-edge, no AppBar, no SafeArea chrome needed).
- Centered `TraevyLogoMark()` (default size 56). No spinner, no text, no progress indicator ‚Äî this is a flash-frame during Firebase session restore, typically sub-second.
- Rendered **inline** from `app.dart`'s auth-gate `switch (auth) { AuthLoading() => SplashScreen ... }`. Not a named route, not pushed onto the navigator.
- Must respect light/dark (the logo mark already inverts via `onSurface`/`scaffoldBackgroundColor`).

### B. Sign-In Bottom Sheet (D-08 ‚Äî opened from Settings guest row)
- `showModalBottomSheet` with `backgroundColor: colorScheme.surfaceContainerLowest` and `showDragHandle: true` ‚Äî **match the existing theme-picker sheet** in `settings_screen.dart` exactly (same surface token, same drag handle, dismissable by drag/scrim tap).
- `SafeArea` + `Column(mainAxisSize: min)`. Content padding: horizontal **20px**, with **24px** top breathing room below the drag handle and bottom padding for the home indicator.
- Content order (top ‚Üí bottom):
  1. Leading Google glyph (`Icons.g_translate_rounded`, 20px) ‚Äî small, above or inline with headline.
  2. Headline **"Back up your commutes"** ‚Äî Inter 22px w700.
  3. **12px** gap.
  4. Subtext **"Your trips sync automatically when you sign in."** ‚Äî Inter 16px w400, `textDim`, line-height 1.5.
  5. **24px** gap.
  6. **Reuse `GoogleContinueButton`** with `onTap` wired to `AuthService.signIn()` (do not build a second button widget).
- Dismissable (D-08). On successful sign-in: dismiss the sheet; the Settings Account section re-renders to the populated `AccountRow` via `authStateProvider` (no confirmation screen when signing in from Settings ‚Äî the one-time confirmation is the onboarding-first-sign-in path; from Settings the row simply updates).
- **Disabled state (D-15):** when `firebaseReady == false`, the embedded `GoogleContinueButton` is disabled (38% opacity, no ripple) with the "Sign-in not configured" tooltip.

### C. Post-Sign-In Confirmation Screen (D-12 ‚Äî one-time, first sign-in via onboarding)
- Full-screen `Scaffold` with `bg` background, `SafeArea`, horizontal **28px** padding (match onboarding screen rhythm).
- Vertically centered content block:
  1. Avatar circle ‚Äî **44x44dp** (or scaled up to 56 for hero emphasis is acceptable), `accentBg` fill + `accent` initial letter ‚Äî same treatment as `AccountRow` avatar. If a Google photo URL is available it may fill the circle; otherwise the first-letter initial.
  2. **48px** gap (hero breathing room).
  3. Headline **"You're signed in."** ‚Äî Inter 36px w700, letter-spacing -1.2, line-height 1.05 (display style, matches onboarding headline).
  4. **12px** gap.
  5. Body **"Your commutes will back up automatically."** ‚Äî Inter 16px w400, `textDim`, line-height 1.5.
  6. `Spacer` pushing the CTA to the bottom (match onboarding layout pattern).
  7. CTA **"Let's go"** ‚Äî reuse the neutral button treatment (`bgElev` fill, `borderStr` outline, Inter 14px w600, 14px radius, `horizontal:18 vertical:16` padding ‚Äî identical shell to `GoogleContinueButton` minus the Google glyph). Tap ‚Üí MainShell.
- **One-time only:** shown only when `AuthService.signIn()` reports a first sign-in (backfill changed rows, per RESEARCH Pattern 4). Never shown again. Not a persistent route in normal navigation.

### Wired-only surfaces (NO visual change ‚Äî confirm contract, don't rebuild)
- **Onboarding `GoogleContinueButton`:** wire `onTap: () => ref.read(authServiceProvider).signIn()`. Visual unchanged.
- **Onboarding "Skip" link:** already routes to MainShell as guest (D-05). No visual change.
- **Settings Account section (D-07):** becomes `ConsumerWidget` watching `authStateProvider`.
  - **Guest:** replace `AccountRow` with a single tappable row ‚Äî leading Google glyph + label **"Sign in to back up"** (Inter 15px w600), chevron affordance, tap ‚Üí opens sign-in sheet (B). Use the existing `SettingsRow` shell where possible.
  - **Signed-in:** existing `AccountRow` populated with real Firebase `displayName` / `email` / first-letter initial (constructor swap only ‚Äî D-07 "constructor swap, not a widget rewrite").
- **"Sign out" row:** stays a rendered no-op visual (`danger` label). Deferred ‚Äî do not wire.
- **"Cloud sync" / "Restore from cloud" rows:** unchanged (Phase 11). Do not alter.

## Interaction & State Contract

| State | Trigger | UI result |
|-------|---------|-----------|
| `loading` | App boot, Firebase session restoring | Static splash (A) |
| `guest` | No session, or `firebaseReady == false` (D-15) | MainShell; Settings shows "Sign in to back up" row; onboarding CTA available |
| `signedIn` | `currentUser` non-null or successful sign-in | MainShell; Settings shows populated `AccountRow` |
| sign-in success (onboarding, first time) | `signIn()` backfilled rows | Push one-time confirmation screen (C) |
| sign-in success (from Settings) | `signIn()` from sheet | Dismiss sheet; Account row re-renders. No confirmation screen. |
| sign-in cancelled | `GoogleSignInException(canceled)` | Silent no-op; stay guest; dismiss sheet |
| sign-in failed | network/credential error | In-sheet error copy; CTA re-enabled |
| Firebase unconfigured | init threw / no `google-services.json` | Sign-in buttons disabled + "Sign-in not configured" tooltip (dev/test) |

**Accessibility / touch targets:** all tappable rows and buttons keep a minimum 44dp height (existing settings rows + 44dp avatar already satisfy this). Disabled buttons set `Semantics(enabled: false)` and expose the tooltip text.
