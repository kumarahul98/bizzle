# Phase 6 — UI Design Contract

> Visual and interaction contract for the Dashboard home screen (UX-01). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Locked decisions inherit from `.planning/phases/06-dashboard/06-CONTEXT.md`.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3 — shadcn does not apply to Flutter) |
| Preset | not applicable |

- `TripCard` from Phase 4 (`lib/features/trips/widgets/trip_card.dart`) is reused without modification. The card uses `colorScheme.surfaceContainerLow` — match that exact token in all new Phase 6 cards.

**AppBar** (CONTEXT D-02, D-07):
- Title area (left): Today's date formatted as `DateFormat('EEE, d MMM').format(DateTime.now())` — e.g. "Sun, 27 Apr". Generic date with no user name (auth is Phase 8). Source: RESEARCH.md Open Question 3.
- Trailing actions (right): `IconButton(icon: Icons.history, tooltip: 'History')` → `Navigator.pushNamed(kRouteHistory)`; `IconButton(icon: Icons.bar_chart, tooltip: 'Stats')` → `Navigator.pushNamed(kRouteStats)`.
- Trailing action for manual entry (CONTEXT D-08, Claude's discretion): `IconButton(icon: Icons.add, tooltip: 'Add trip manually')` → calls `_handleAddManualTrip`. Placing it as an AppBar icon avoids gesture conflicts with the FAB primary action and is always discoverable. Three trailing icons total; rightmost order: add, history, bar_chart.
- Elevation: default Material 3 `AppBar` (no custom elevation).

**Scrollable body layout** (CONTEXT D-02):
1. `WeeklySummaryCard` — full width, 16px horizontal padding, tappable.
2. Section label — `Text('Today', style: textTheme.titleMedium)` with `kDashboardTodaySectionLabel` constant; 16px horizontal padding, 16px above, 8px below.
3. `TodayTripsSection` — renders `InProgressCard` (if active), then flat `TripCard` list (or empty-state text).
4. `SizedBox(height: 32)` at the end of the scroll list so the last card clears the FAB.

## Spacing Scale

Inherits the Phase 5 contract exactly. Do not introduce new values.

| Token | Value | Usage in Phase 6 |
|-------|-------|------------------|
| xs | 4px | Gap between label and value inside `InProgressCard` |
| sm | 8px | Gap between the "Today" section label and the first card below it; gap between rows inside `WeeklySummaryCard` |
| md | 16px | `Card` internal padding (`EdgeInsets.all(16)`); horizontal screen padding for all body content; vertical gap between `WeeklySummaryCard` and the "Today" label |
| lg | 24px | Top padding of the scroll body (`SliverPadding` or `padding` on `ListView`) |
| xl | 32px | Bottom padding at end of scroll list so the last card does not sit flush against the FAB |
| 2xl | 48px | Reserved — not used in Phase 6 |
| 3xl | 64px | Reserved — not used in Phase 6 |

**Exceptions:**
- FAB touch target: Flutter's `FloatingActionButton.extended` provides at least 48dp height by default — no override needed.
- AppBar trailing `IconButton` touch targets: `IconButton` default is 48dp — no override needed.
- `InProgressCard` uses a left-border accent stripe of **4px width** (`BorderSide.width: 4`) rendered via `Card.shape: RoundedRectangleBorder(side: BorderSide(color: colorScheme.primary, width: 4), ...)` — this is a single styling exception, not a new spacing token.

## Typography

Inherits from `Theme.of(context).textTheme` (Material 3 Roboto roles). Phase 6 declares **exactly 4 sizes / 2 weights** — must not introduce custom `TextStyle` instances except where the table allows. Consistent with Phase 5 typography contract.

| Role | TextTheme token | Size (sp) | Weight | Line height | Phase 6 usage |
|------|----------------|-----------|--------|-------------|---------------|
| Display number | `headlineMedium` | 28sp | w600 (semibold) | 1.25 | Elapsed time value in `InProgressCard`; the two duration values in `WeeklySummaryCard` |
| Heading | `titleMedium` | 16sp | w600 (semibold) | 1.20 | "Today" section label; row labels inside `WeeklySummaryCard` ("This week", "In traffic"); `AppBar` date text |
| Body | `bodyMedium` | 14sp | w400 (regular) | 1.43 | Trip count row in `WeeklySummaryCard` ("3 trips"); empty-state text; `InProgressCard` sub-label |
| Label | `labelMedium` | 12sp | w500 (medium — theme-provided) | 1.33 | FAB label text via `FloatingActionButton.extended` default |

**Weight policy:** only **w400 (regular)** and **w600 (semibold)** may be authored via `copyWith(fontWeight: ...)`. The `labelMedium` w500 is accepted from the theme default, never authored explicitly.

**Duration values** (elapsed time, weekly total, traffic time) MUST use `headlineMedium` and `formatDuration(int seconds)` from `lib/shared/utils/formatters.dart`. No manual string concatenation.

**Trip count pluralization** (Claude's discretion): `'1 trip'` when count == 1, `'$count trips'` otherwise. Use `kDashboardTripCountSingular` and `kDashboardTripCountPlural` constants; build the string with string interpolation at call site (not hardcoded).

## Color

Phase 6 reads every color from `Theme.of(context).colorScheme`. Never use raw hex values. Consistent with Phase 5 color contract and `TripCard`'s existing usage.

| Role | ColorScheme token (light) | ColorScheme token (dark) | Phase 6 usage |
|------|---------------------------|--------------------------|---------------|
| Dominant (60%) | `surface` | `surface` | `Scaffold` background; page surface behind all cards |
| Secondary (30%) | `surfaceContainerLow` | `surfaceContainerLow` | `WeeklySummaryCard` card background; `InProgressCard` card background — matches existing `TripCard` at `trip_card.dart:39` |
| Accent (10%) | `primary` (and `primaryContainer`, `onPrimaryContainer`) | `primary` | Reserved exclusively for: (a) `InProgressCard` left-border stripe (`colorScheme.primary`, 4px), (b) FAB fill color (Material 3 default for `FloatingActionButton` — `colorScheme.primaryContainer`) |
| Destructive | `error` (and `onError`) | `error` | Reserved exclusively for delete confirmation dialogs inherited from `TripCard`'s existing options sheet. **Not introduced by Phase 6** — `TripCard` already uses `colorScheme.error` for the delete list tile icon. |
| Sub-text / disabled | `onSurfaceVariant` | `onSurfaceVariant` | Empty-state text; row sub-labels inside cards; direction icon in `TripCard` (existing) |

**Accent reserved for (explicit list — checker dimension 3):**
1. `InProgressCard` left-border stripe — `colorScheme.primary`, width 4px, via `Card.shape` `BorderSide`.
2. `FloatingActionButton.extended` fill — `colorScheme.primaryContainer` (Material 3 default, not authored).

**Forbidden in Phase 6:**
- Custom hex colors (`Color(0xFF...)`) — `colorScheme` tokens only.
- Tertiary, `surfaceTint`, or any M3 token not listed above.
- Color-only differentiation — `InProgressCard` must include the `Icons.timelapse` icon alongside the accent border so tracking state is not communicated by color alone (WCAG 1.4.1).

## Interaction Contract

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Start commute (idle) | Tap FAB ("Start commute") | Calls `_handleStart(context, ref)` — migrated verbatim from `HomeScreen`; checks permissions, navigates to `kRouteTracking` or shows settings dialog |
| Go to tracking (active) | Tap FAB ("Go to tracking") | `Navigator.pushNamed(context, kRouteTracking)` directly — no permission check when already tracking |
| Add manual trip | Tap AppBar add `IconButton` | Calls `_handleAddManualTrip(context, ref)` — migrated verbatim from `HomeScreen`; opens `ManualEntrySheet` bottom sheet |
| Open history | Tap AppBar history `IconButton` | `Navigator.pushNamed(context, kRouteHistory)` |
| Open stats | Tap AppBar stats `IconButton` | `Navigator.pushNamed(context, kRouteStats)` |
| Open stats via card | Tap anywhere on `WeeklySummaryCard` | `Navigator.pushNamed(context, kRouteStats)` — wrap card in `InkWell` with `borderRadius` matching `Card` default |
| Open tracking in-progress | Tap `InProgressCard` | `Navigator.pushNamed(context, kRouteTracking)` |
| Open trip detail | Tap `TripCard` body | `Navigator.pushNamed(context, kRouteTripDetail, arguments: trip.id)` — handled by `TripCard` internally |
| Trip options | Tap `TripCard` more-vert icon | Opens bottom sheet with Edit / Delete — handled by `TripCard` internally |
| Scroll | Vertical drag | Standard Material scroll via `ListView` or `CustomScrollView`; no parallax, no sticky headers |
| System back | Hardware/gesture back | No-op on dashboard (it is the app root; `WillPopScope` / `PopScope` not required — Android handles root gracefully) |

**FAB behavior detail (CONTEXT D-03):**
- FAB is ALWAYS visible — never hidden regardless of scroll position. Do NOT use `floatingActionButtonLocation: FloatingActionButtonLocation.endFloat` with any scroll-driven hide logic.
- `trackingStateProvider` is watched once in `DashboardScreen.build()`. The result is passed down to child widgets — children do NOT watch `trackingStateProvider` independently.
- FAB mode switch: `trackingState is TrackingActive` → active mode; all other states (idle, loading, error) → idle mode.

**Loading state behavior:** `AsyncValue` state is consumed at `DashboardScreen` level via `todaysTripSummariesProvider` and `statsSummaryProvider`. Cards must NOT flicker between loading and data on individual Drift emissions — both providers use `whenData` derivation that preserves the previous value while recomputing. Render `CircularProgressIndicator` only on the initial load (before first emission).

**`WeeklySummaryCard` InkWell:** The entire card surface is tappable. Wrap in `InkWell(onTap: ..., borderRadius: BorderRadius.circular(12))` inside the `Card`. Do NOT add a ripple to individual rows inside the card.

## Accessibility

| Rule | Implementation |
|------|----------------|
| Touch target ≥ 44dp | `FloatingActionButton.extended` is 56dp; `IconButton` is 48dp (Material defaults — no override needed) |
| Color is not the only signal | `InProgressCard` uses `Icons.timelapse` icon alongside the `colorScheme.primary` border stripe; FAB switches both icon and label (not just color) between states |
| Screen reader labels | AppBar `IconButton`s use `tooltip:` parameter (maps to `Semantics.label`). `InProgressCard` wraps its content in `Semantics(label: 'Commute in progress, elapsed: $elapsed')`. `WeeklySummaryCard` rows use semantic ordering (title read before value). |
| Contrast | Inherits Material 3 default scheme — WCAG AA in light, AAA in dark for body text. Do NOT override `onSurface` / `onSurfaceVariant` text colors with custom values. |
| Text scaling | Use `TextTheme` tokens via `Theme.of(context).textTheme` — never pass `TextStyle(fontSize: ...)` directly. Cards must reflow vertically when `textScaler` ≥ 1.5 (use `Column`, not `Row`, for card content rows). |

## Claude's Discretion (resolved)

CONTEXT.md flagged these as Claude's discretion. All resolved here so the planner has a fixed contract:

1. **AppBar title/greeting text:** Today's date formatted as `DateFormat('EEE, d MMM').format(DateTime.now())` (e.g. "Sun, 27 Apr"). No user name — auth is Phase 8. Provides immediate informational value over a static "Traevy" label. Source: RESEARCH.md Open Question 3.

2. **"In progress" card visual treatment:** `Card` with `colorScheme.surfaceContainerLow` background, a 4px `colorScheme.primary` left-border stripe via `Card.shape`, and a `Icons.timelapse` leading icon in `colorScheme.primary` color. Title: `kDashboardInProgressLabel` ("In progress") in `titleMedium`. Elapsed time in `headlineMedium` via `formatDuration`. Subtitle: distance in km (1 decimal place) using `(active.distanceMeters / 1000).toStringAsFixed(1)` + `' km'`. Tapping navigates to tracking screen.

3. **"In traffic" vs "Stuck in traffic" label:** Use `kDashboardInTrafficLabel` = `'In traffic'` (shorter form). The weekly summary card has constrained vertical space; the full phrase is redundant when the card is already in context.

4. **Empty state label text:** `kDashboardEmptyStateLabel` = `'No commutes yet today'`. Centered, `bodyMedium` style, `colorScheme.onSurfaceVariant` color. No illustration, no CTA — the FAB is always visible as the natural next step.

5. **Trip count label pluralization:** `kDashboardTripCountSingular` = `'1 trip'`; `kDashboardTripCountPlural` = `'$count trips'` (executor builds with `'$count trips'` interpolation at call site using the int from `todaysTripSummariesProvider`). The count is TODAY's trip count, not the full week count (CONTEXT D-06 verbatim).

6. **Manual entry placement (D-08):** AppBar trailing `IconButton` with `Icons.add` and `tooltip: 'Add trip manually'`. Three trailing icons total — ordered right-to-left in the AppBar: bar_chart (rightmost), history, add (leftmost of the three).

7. **Weekly summary trip count scope (RESEARCH.md Open Question 1):** Implement as today's count from `todaysTripSummariesProvider`, verbatim per CONTEXT D-06. The card is titled "This week" for the duration rows but the count row deliberately shows today's count as a quick "how many commutes today" summary. This is the decision from CONTEXT; if it looks wrong during QA, escalate to user.

8. **File and widget naming:** See Component Inventory table above. Exact names are locked by this spec.

9. **Provider naming:** `todaysTripSummariesProvider` (declared in RESEARCH.md Pattern 1).

## Acceptance Criteria (mirrors UX-01)

- [ ] **UX-01 summary card:** `WeeklySummaryCard` renders "This week" total duration, "In traffic" duration, and today's trip count ("N trips / 1 trip"). Tapping the card navigates to Stats. All values from `statsSummaryProvider` + `todaysTripSummariesProvider`.
