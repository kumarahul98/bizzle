## Spacing Scale

Declared values (multiples of 4 — extends the Phase 4 contract; do not introduce new values in Phase 5):

| Token | Value | Usage in Phase 5 |
|-------|-------|------------------|
| xs | 4px | Compact gap inside chip rows (between best/worst day chips when wrapping is forced) |
| sm | 8px | Gap between a card label and its value row (e.g., "This week" → "2h 34m") |
| md | 16px | Default `Card` internal padding (`EdgeInsets.all(16)`); horizontal screen padding for list-style content; gap between consecutive `StatsCard` widgets |
| lg | 24px | Vertical padding inside `TrendChartCard` so the `fl_chart` plot does not collide with the card label; empty-state heading gap (matches `_kEmptyHeadingGap` in `history_screen.dart`) |
| xl | 32px | Bottom safe-area padding at the end of the scroll list so the last card is not flush against the gesture bar |
| 2xl | 48px | Reserved — not used in Phase 5 |
| 3xl | 64px | Reserved — only used by full-screen empty-state icon if a global empty state is ever added (Phase 5 does NOT use this; D-10 prohibits a full-screen empty state) |

**Exceptions:**
- `TrendChartCard` plot area has a fixed height of **192px** (multiple of 4, matches Phase 4 `kTripDetailMapHeight = 256` proportions but smaller — chart needs less real estate than a map). Declared as `kStatsTrendChartHeight` in `lib/config/constants.dart`.
- `BestWorstDayCard` day chip min touch target is **44px** (Material accessibility spec, allowed exception per template) — achieved via `Chip(visualDensity: VisualDensity.standard)` not `compact`.

## Typography

Inherits from `Theme.of(context).textTheme` (Material 3 Roboto roles). Phase 5 declares **exactly 4 sizes / 2 weights** — must not introduce custom `TextStyle` instances except where the table allows.

| Role | TextTheme token | Size (sp) | Weight | Line height | Phase 5 usage |
|------|----------------|-----------|--------|-------------|---------------|
| Display number | `headlineMedium` | 28sp | w600 (semibold) | 1.25 | The big number in each card: weekly total, monthly total, traffic waste, direction averages |
| Heading | `titleMedium` | 16sp | w600 (semibold) | 1.20 | Card label (e.g., "This week", "Time stuck in traffic"); `AppBar` title |
| Body | `bodyMedium` | 14sp | w400 (regular) | 1.43 | Secondary text inside cards (e.g., "Mon–Sun", direction sub-labels); empty-state body copy |
| Label | `labelMedium` | 12sp | w500 (medium) — see note | 1.33 | Day-of-week chip labels (Mon, Tue, …); chart axis labels (Week 1, Week 2, Week 3, This week) |

**Weight policy:** only **w400 (regular)** and **w600 (semibold)** are allowed. Material 3's `labelMedium` ships at w500; we accept that as the chip default because (a) it is theme-provided, not declared by us, and (b) it is the only place w500 appears, never authored. Custom `TextStyle.copyWith(fontWeight: ...)` calls in Phase 5 may use ONLY `FontWeight.w400` or `FontWeight.w600`.

**Numeric values inside cards** (durations like "2h 34m") MUST use `headlineMedium` and the duration formatter from `lib/shared/utils/formatters.dart` (Phase 4 `formatDuration`) — no manual string concatenation.

**Empty-state placeholder**: when a stat is unavailable (D-10), render `—` (en-dash, U+2014 / em-dash style) in the same `headlineMedium` slot the number would have occupied. Do NOT shrink the type.

## Color

The app currently uses Material 3 default `ColorScheme` from `ThemeData.light(useMaterial3: true)` / `ThemeData.dark(useMaterial3: true)`. Phase 5 must read every color from `Theme.of(context).colorScheme` — never use raw hex values.

| Role | ColorScheme token (light) | ColorScheme token (dark) | Phase 5 usage |
|------|---------------------------|--------------------------|---------------|
| Dominant (60%) | `surface` | `surface` | `Scaffold` background, default page surface behind cards |
| Secondary (30%) | `surfaceContainerLow` | `surfaceContainerLow` | Stat `Card` background — matches existing `TripCard` (`trip_card.dart:39`) |
| Accent (10%) | `primary` (and its `onPrimary`/`primaryContainer` variants) | `primary` | Reserved exclusively for: (a) the BEST day chip's selected indicator, (b) the trend `LineChart` line stroke, (c) the trend chart's data-point dots |
| Destructive | `error` (and `onError`) | `error` | Reserved exclusively for: the WORST day chip's selected indicator. **Not used as an action color in Phase 5** — there are no destructive actions on this screen. |
| Sub-text / disabled | `onSurfaceVariant` | `onSurfaceVariant` | "—" empty placeholder, secondary card sub-labels, day-chip text for non-best/worst days |

**Accent reserved for (explicit list — checker dimension 3):**
1. `BestWorstDayCard` best-day chip background (`colorScheme.primaryContainer`) and its label color (`colorScheme.onPrimaryContainer`).
2. `TrendChartCard` `LineChartBarData.color` (the trend stroke).
3. `TrendChartCard` `FlDotCirclePainter.color` (the trend data dots).

**Destructive reserved for:**
1. `BestWorstDayCard` worst-day chip background (`colorScheme.errorContainer`) and its label color (`colorScheme.onErrorContainer`). No `error` swatch elsewhere.

**Forbidden in Phase 5:**
- Custom hex colors (`Color(0xFF...)`) — read from `colorScheme` only.
- Tertiary, surfaceTint, or any other M3 token not listed above.
- Color-only differentiation for best/worst — chips MUST also include a leading icon (`Icons.trending_down_rounded` for best, `Icons.trending_up_rounded` for worst) so the screen reads correctly for color-blind users (WCAG 1.4.1).

Reuse the `intl` `DateFormat.E(locale).format(date)` for short weekday names so locale changes propagate automatically. **Do not** hardcode "Mon" / "Tue" strings — derive them. Tests pin the locale to `en_US` to assert "Mon", "Tue", "Wed", "Thu", "Fri".

### Empty / loading / error states

| State | Copy | Treatment |
|-------|------|-----------|
| Loading (StreamProvider `loading`) | (no copy) | `Center(child: CircularProgressIndicator())` in the body slot. Same pattern as `HistoryScreen:95`. |

## Interaction Contract

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Open Stats | Tap "View stats" `OutlinedButton` on home screen | `Navigator.pushNamed(context, kRouteStats)` |
| Scroll | Vertical drag on the body | Standard Material scroll; no parallax, no sticky headers (single page, not grouped) |
| Tap a stat card | (no-op in Phase 5) | Cards are read-only. No deeper drill-down screen exists. Do **not** add `InkWell` ripples — they signal interactivity that does not exist. |
| Tap a day chip in best/worst card | (no-op in Phase 5) | Chips are display-only. Do **not** wrap them in `ActionChip` or `FilterChip`. Use plain `Chip`. |
| Tap a trend chart point | (no-op in Phase 5) | `fl_chart` `lineTouchData` MUST be set to `LineTouchData(enabled: false)`. Interactivity is deferred to a future phase (CONTEXT deferred §3). |
| System back | Hardware/gesture back | Standard `Navigator.pop` — returns to home screen |

**Loading state behavior**: `StreamProvider` re-emits on every Drift change. Cards must NOT flicker between value and loading on every emission — use `AsyncValue.when` at the screen root only, not per-card. When data arrives, all cards transition from `CircularProgressIndicator` to populated values in one frame.

## Accessibility

| Rule | Implementation |
|------|----------------|
| Touch target ≥ 44dp | Day chips use `VisualDensity.standard` (not compact); home-screen "View stats" button inherits Material's 48dp default |
| Color is not the only signal | Best/worst day chips include leading icons (`Icons.trending_down_rounded` / `Icons.trending_up_rounded`); trend chart line stroke is supplemented by axis labels |
| Screen reader labels | `Semantics(label: 'Best commute day: Tuesday, average 24 minutes')` on best/worst chips; `Semantics` wrapper on each card so VoiceOver/TalkBack reads `<title> <value> <helper>` in order |
| Contrast | Inherits from Material 3 default scheme — WCAG AA in light, AAA in dark for body text. Phase 5 must NOT override `onSurface`/`onSurfaceVariant` text colors. |
| Text scaling | Use `MediaQuery.textScaler` implicitly via `TextTheme` — never pass fixed `TextStyle(fontSize: ...)`. Cards must reflow vertically when textScaler ≥ 1.5 (Wrap, not Row, for chip rows). |

## Claude's Discretion (resolved)

CONTEXT.md flagged these as "Claude's discretion". Resolved here so the planner has a fixed contract:

1. **Trend chart days with no trips**: render as `0` value (flat baseline), NOT a visual gap. `fl_chart` handles this naturally; gaps require null-handling that complicates the data pipeline. Empty days at the start of the 28-day window simply produce a flat lead-in, which truthfully conveys "no commutes recorded".
2. **Chart widget type**: `LineChart` (per CONTEXT specifics §4). No `BarChart`, no `ScatterChart`.
3. **Best/worst visual highlight**: filled chip with `primaryContainer`/`errorContainer` background + matching `on*` text + leading icon. Other days are unstyled `Chip` with default `surfaceContainerHighest` background. Best is always green-coded (primary), worst is always red/amber-coded (error).
4. **File naming inside `lib/features/stats/`**: `screens/stats_screen.dart`, `widgets/stats_card.dart`, `widgets/week_month_totals_card.dart`, `widgets/direction_averages_card.dart`, `widgets/best_worst_day_card.dart`, `widgets/trend_chart_card.dart`, `widgets/traffic_waste_card.dart`, `providers/stats_providers.dart`.
5. **Provider granularity**: ONE `StreamProvider<StatsSummary>` (per CONTEXT D-06) emitting an immutable `StatsSummary` record/class with all 5 stats. Cards read fields off the same async value. No per-card providers.
6. **AppBar title**: `Stats` (single word, matches "History").
7. **Empty-card label text**: `kStatsEmptyPlaceholder = '—'` only — no longer "—" plus a trailing word like "no data". One symbol keeps cards visually quiet while no trips exist.

### Open question for the user (single confirmation needed)

The `kStatsErrorMessage` references "pull down to retry" but Phase 5 does not implement `RefreshIndicator`. Two options:

- **Option A (recommended):** drop the retry hint — copy becomes `Could not load stats.` since the StreamProvider auto-recovers when Drift emits the next change.
- **Option B:** wrap the body in `RefreshIndicator` and have it call `ref.invalidate(statsProvider)` — costs ~10 lines of code, gives users a manual escape hatch.

Defaulting to **Option A** (less code, matches existing `HistoryScreen` error treatment which also has no pull-to-refresh). Planner can flip to Option B during planning if the user disagrees.

## Acceptance Criteria (mirrors STAT-01..05)

The implemented screen passes when:

- [ ] **STAT-01**: `WeekMonthTotalsCard` displays both weekly (Mon–Sun containing today) and monthly (1st–last of current calendar month) total commute time, formatted via `formatDuration`. Empty → `—`.
- [ ] **STAT-02**: `DirectionAveragesCard` shows two rows — "To office" and "To home" — each with the average duration for that direction across all trips. Empty per-direction → `—`.
- [ ] **STAT-03**: `BestWorstDayCard` shows 5 chips (Mon–Fri); the day with the lowest average duration has the primary-tinted chip + downward-trend icon, the highest has the error-tinted chip + upward-trend icon. Saturday and Sunday are excluded. With zero trips, all 5 chips render unstyled (no best/worst highlight).
- [ ] **STAT-04**: `TrendChartCard` renders an `fl_chart LineChart` over the last 28 calendar days with a single combined line; x-axis = "Week 1", "Week 2", "Week 3", "This week"; y-axis = minutes (auto-scaled); zero-trip days produce 0-value points (flat segment). Empty → chart axes still render with all-zero baseline.
- [ ] **STAT-05**: `TrafficWasteCard` displays the sum of `timeStuckSeconds` for all non-manual trips in the current week, formatted via `formatDuration`. Manually-entered trips are excluded (D-05). Empty → `—`.
- [ ] **D-01**: All 5 cards stack vertically in the order specified, padding 16 between cards, 16 horizontal screen padding.
- [ ] **D-02**: Home screen has "View stats" `OutlinedButton` placed below "View history" with the same width factor (0.7) and 12px gap.
- [ ] **D-10**: With zero trips, every card renders structurally intact with `—` in each value slot — no full-screen empty state.
- [ ] **a11y**: All chips have `Semantics` labels including best/worst designation and underlying duration. No color-only signalling.
- [ ] **Constants**: every visible string and numeric layout value lives in `lib/config/constants.dart`. No inline string literals in widget files except for cross-feature reuse (`'/stats'` route — already in `routes.dart`).
