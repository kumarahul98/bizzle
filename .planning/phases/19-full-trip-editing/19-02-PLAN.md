<objective>
Extend the trip edit sheet into the full-editing UI: date+time pickers for start/end (D-09), an add/edit/remove break-list editor (extracted into break_editor_list.dart + break_row.dart, each <100 lines), a live in-memory recompute preview of duration + moving/stuck (D-11), inline validation messages with a disabled Save until valid (D-08), the clamp/drop "breaks adjusted" snackbar (D-10), and a subtle "~ estimated" hint on the trip detail + history row when a trip is_edited (D-04). On Save the sheet computes the recomputed stats via `TripEditRecompute` and calls the extended `editTrip` full-edit path from Plan 01, which persists atomically and re-enqueues sync (TRACK-11, SC#1/2/3/4/5).

Purpose: Plan 01 built the pure recompute/validation logic + the atomic write; this plan wires it to the user. The sheet is the single surface where every time-based detail becomes editable. Validation is in-memory and gates Save so impossible edits never reach Drift (D-08/D-11). The "~ estimated" hint keeps us honest that edited moving/stuck are derived, not measured (D-04).
Output: the extended sheet + three new <100-line widgets + the estimated hint + the hint wired into trip detail/history row, with widget tests for editing start/end, add/remove break, invalid-blocks-save, save-persists-and-shows-recompute, and estimated-hint-appears.
</objective>
