Recompute math (D-01/D-02) — implement as pure functions, no Drift:
```text
activeSeconds(start, end, breaks) = (end - start).inSeconds - Σ (b.end - b.start).inSeconds
  // breaks here are the validated/clamped, closed segments
rescale(origMoving, origStuck, newActive):
  total = origMoving + origStuck
  if total == 0 -> (moving: 0, stuck: 0)          // D-02, never invent a ratio
  newMoving = (newActive * origMoving / total).round()
  newStuck  = newActive - newMoving               // exact complement so sum == active
  // clamp newMoving into [0, newActive] before computing newStuck (guard rounding at 0-length)
```

Validation rules (D-05/06/07) operate on a List of (start,end) break segments in WALL-CLOCK UTC:
- each: start < end strictly (D-06)
- each: start >= trip.start AND end <= trip.end (boundary-touch allowed) (D-05)
- sorted by start, for every adjacent pair: prev.end < next.start STRICTLY (reject overlap AND touch) (D-07)
Return a structured result naming the FIRST failing rule so Plan 02 can show a specific inline message (use constants below, never hardcoded English in the service).

Clamp/drop (D-10) — applied when the trip WINDOW shrinks, BEFORE validation:
- break fully outside [newStart, newEnd] (end <= newStart || start >= newEnd) -> DROP
- break partially outside -> CLAMP start = max(start, newStart), end = min(end, newEnd)
- if a clamp makes start >= end -> DROP that break
- report `adjusted = true` if any break was dropped or clamped (Plan 02 shows the snackbar)
