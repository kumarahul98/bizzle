### Real bugs found — both FIXED & regression-tested (2026-07-18)

Update: both bugs below were fixed the same cycle they were found (two parallel single-module agents, central verification, full suite 653 green — up from 647). **BUG-1 → commit `452afd8`; BUG-2 → commit `c22a2aa`.** Details retained below for the record.


**BUG-1 — TRACK-13 interrupted-trip recovery is non-functional in production (Phase 25, `gaps_found`, 1/5).**
`TripStatePersister` is never injected into `TripAccumulator` at any of the four production construction sites — `main_isolate_tracking_engine.dart:119,121` and `tracking_service.dart:85,88` (the real Android background isolate). With `_persister == null`, `TripAccumulator._persistState()` (`trip_accumulator.dart:251`) early-returns on every sample/pause/resume, so `active_trip.json` is never written during a real trip. Launch-time detection (`tracking_providers.dart:190` `loadState()`) therefore always returns null; the recovery prompt, resume, and discard paths are all unreachable. Every surrounding layer (persister, serializer, sealed `TrackingInterrupted` state, dialog, resume plumbing) is correct and unit-tested — CI passes because tests inject a fake persister; production never does. **Fix (small):** pass `TripStatePersister()` at the four sites (incl. on `restore(...)` so a recovered trip keeps persisting) + a test that constructs a production-shaped accumulator and asserts it writes on addSample/pause/resume. Tracking id: **PERSIST-INJECT-FIX**. TRACK-13 traceability set to **Blocked**.

**BUG-2 — LOC-02 historical backfill never runs (Phase 21, part of `gaps_found` 5/6).**
`geofenceBackfillProvider` is only ever `ref.invalidate`d (`location_picker_screen.dart:119`) and is never `watch`/`read`/`listen`ed anywhere — invalidating a never-listened keepAlive `FutureProvider` is a no-op, so its body never executes. The working sibling `directionBackfillProvider` is `ref.watch`'d at `app.dart:50`; the geofence one has no consumer. **New-trip auto-labeling at finalize is unaffected and fully verified** (this is the LOC-02 requirement text, so LOC-02 stays Complete), but the historical re-label of pre-existing trips after first setting Home/Office is dead. **Fix (small):** `await ref.read(geofenceBackfillProvider.future)` on the picker's confirm path + a test proving saving a location runs the service. Tracking id: **GEO-BACKFILL-FIX**.

### Why the 2026-07-13 integration pass missed both

The cross-phase integration checker traced **export → consumer declarations** (does A import/reference B?) and both bugs pass that test: the persister is imported and the backfill provider is referenced. It did not trace **runtime activation** (is the dependency actually injected / is the provider actually subscribed?). Both bugs live precisely in that gap — a declared link that never fires at runtime. Lesson for future audits: "wired" from an integration checker is necessary but not sufficient; goal-backward verification that follows real data flow is what caught these.
